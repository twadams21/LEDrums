- Triggers: boot (before any mutation), 30-min cadence gated on content hash, pre-risk (schema migrations, bulk project applies/setProject, before restore). Pre-risk trigger list is append-only.
- Local retention: 48 recent + one/day for 30 days; pre-risk on a separate fixed budget of 20.
- Off-site: second disk-backed queue → backup route on the #122 Worker → R2 under `backups/<machine>/<timestamp>-<reason>`; R2 lifecycle expiry 90 days (config, not code — document it in the Worker README). Read API: list by machine + fetch by key, same bearer token.
- Wire surface: two new WS client messages — list backups, restore-by-id. Restore: pre-risk snapshot first → atomic replace of all three blobs → engine/show reload → broadcast like a cold load. v1 restore is local-only.
- UI: a **Backups dialog** following the ShowBrowser dialog pattern — snapshot list (relative time + reason) + Restore with confirm; confirm copy states a backup of current state is taken first.
