# S45 — Patch copy/paste: setProject + diff dialog (group K, #55) — implementer report

Branch `slice/S45` off `group/K` (S43 clipdoc foundation in-tree). Assignment:
`docs/handoff/rock-solid/S45-assignment.md`. Copy-patch serializes the device Project slices
(kit incl. outputs, input map, output settings) as a `patch` ClipDoc; paste applies via a new
bulk `setProject` message — schema-validated server-side, applied ONCE (single kit reload, no
granular replay), persisted, broadcast — behind a diff-confirm dialog. `packages/core` stays pure.

## Acceptance — all met
- [x] Patch round-trips across servers; engine transmits after ONE apply — server test (fakes):
      validate → apply-once → persist (`markDirty`) → broadcast; voice host bulk-adopts same slices.
- [x] Invalid payload rejected with a user-visible error, ZERO partial apply — `projectPatchSchema.safeParse`
      before any state touch → `error` reply to sender + monitor `error`; store surfaces a dismissible notice.
- [x] Diff dialog shows the change summary (drums, pixels, output host, protocol) + requires explicit
      confirm — pure `diffPatch` (tested) → `PatchDiffDialog`; `setProjectPatch` only fires on confirm.

## Layers
- **core** `projectPatchSchema`/`ProjectPatch`/`parseProjectPatch` (kit+inputMap+output+name?).
