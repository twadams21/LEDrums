# S05 — MIDI device list in settings (B, issue #47)

## Summary
The Settings dialog now lists connected WebMIDI input devices. A new "MIDI devices"
section shows each input's name and a Connected/Disconnected status pill, and it
**refreshes on hot-plug without a reload** — plug a controller in and it appears; unplug
it and the row greys to "Disconnected" (the port lingers per the WebMIDI spec) rather than
vanishing. When WebMIDI is unavailable the list is replaced by an empty state that explains
the cause: an unsupported browser vs. a blocked MIDI permission (with a reload hint). When
the API is available but nothing is plugged in, it invites connecting one.

## Context pack
No direct dependent slice. Small reusable seams other IO-confidence slices may build on:
- `webmidi.ts` now has a pure `enumerateDevices(access) → MidiDeviceInfo[]` and an
  `onDevices` callback on `initMidi` (fires on init + every `statechange`). `MidiInitResult`
  gained `devices`; the legacy `inputs: string[]` is retained for back-compat.
- Store exposes reactive `midiDevices` / `midiAvailable` / `midiUnavailableReason` (reset on
  `stop()`). Any surface wanting a live device list can read these.
- Empty-state copy is a pure `deviceListEmptyState(available, reason, count)` helper —
  reuse it if another device list (OSC ports, controllers) needs the same treatment.

## Acceptance
- [x] **Devices listed; plug/unplug updates without reload** — `AppSettingsDialog`
      renders `store.midiDevices` (name + `StatusPill`); the list is fed by `initMidi`'s
      `onDevices` callback which re-publishes the snapshot on every WebMIDI `statechange`.
      Proven by `webmidi.test.ts › re-publishes the device list on hot-plug and unplug`
      (adds a second port on statechange, then flips the first to `disconnected`).
- [x] **Empty state explains browser permission when WebMIDI unavailable** —
      `deviceListEmptyState` returns a browser-support message for `reason:'no-api'` and a
      "MIDI access is blocked — allow permission, then reload" message otherwise; the
      component swaps the list for this copy whenever `midiEmpty` is non-null. Covered by
      `midi-devices.test.ts` (no-api, denied, no-reason, available-but-empty, has-devices).
