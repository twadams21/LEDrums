# S03 — Output status panel (B, issue #47)

## Summary
The Patch controller inspector now opens with a live **Output status** panel — the app's
first ambient surface for "is the pixel stream actually leaving the box." It shows the
server's arming state (Armed / Dry-run / Disabled) as a toned pill, **packets/s** (derived
from the change in `OutputStatus.packetsSent` between `stats` ticks), universe count,
target `host:port`, protocol, and — prominently, as a red `role="alert"` callout — the
`lastError`. Offline, it shows an honest empty hint; on a link drop the truth is cleared
rather than frozen. The panel is a pure presentational composite (plain props) so it demos
in the styleguide and is the extension seam for the PixLite panel (S48).

## Context pack (for S48 — PixLite controller panel EXTENDS this panel)
**The panel is the seam. Extend it; don't rebuild it.** Pointers + invariants:

- **`OutputStatusPanel.svelte`** (`apps/web/src/lib/app/docks/inspectors/`) is a **pure,
  store-free composite**: props `{ output: OutputStatus | null, packetsPerSec: number | null,
  port?: number }`. S48 should add the adopted controller's rx stats (per-universe
  `timedOut`/`inGood`, `inFrmRate`/`outFrmRate`, temp/voltage/port health) + **Identify** and
  **Adopt-IP** controls **below the `.fault` block, inside this component** — e.g. add an
  optional `controller?: ControllerStatus` prop and a new `<section>`. Keep it prop-driven so
  the styleguide demo keeps working.
- **Do NOT overload `store.output`.** The controller's own status is a *different* truth
  (out-of-band, from the PixLite mgmt API) — give it its own store field (`store.controllerStatus`)
  fed by the new `{ t:'controllerStatus' }` WS message. `store.output` is strictly the server's
  send-side `OutputStatus`.
- **Reuse the pure derivations** in `output-status.ts` (`packetsPerSecond`, `formatPacketsPerSecond`,
  `outputStateTone`, `outputStateLabel`, `defaultPort`) — don't reimplement rate math or tone maps.
- **Invariants to preserve:** packets/s `null` → shown as `"—"`; a steady `0/s` is shown
  honestly ("armed but nothing flowing"); the rate resets to `null` on a counter reset (server
  restart / re-arm) and on link drop. The fault callout is the app's red error family
  (hue-25 — cf. `PinGate`, Monitor `.type-error`) and carries `role="alert"`.
- **Wiring:** `PatchControllerInspector.svelte` renders `<OutputStatusPanel/>` + `<Separator/>`
  at the **top** (status-first), above the transport-config fields; `port` is passed from
  `project.output.port` because `OutputStatus` does not carry a port on the wire.
- **Test seams:** `output-status.test.ts` (pure, table-driven) and `OutputStatusPanel.test.ts`
  (`@testing-library/svelte`, jsdom) — extend these rather than starting new files.
