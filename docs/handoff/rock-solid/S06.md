## Context pack for S07 + S08
- **Module:** `apps/web/src/lib/app/desktop-bridge.svelte.ts`. **Reducer + types:** `apps/web/src/lib/app/boot-reducer.ts`.
- **Singleton:** `import { desktopBridge } from '../desktop-bridge.svelte'`. Call `desktopBridge.start()` ONCE at app boot — **S06 never calls it** (no UI change), so S07/S08 must wire that up.
- **Reactive surface** (`class DesktopBridge`): `get bootStatus(): BootStatus` (reactive `$state` getter — read it directly in a `.svelte` component), `get isDesktop(): boolean`, `checkForUpdate(): Promise<UpdateCheckResult|null>` (folds availability into `bootStatus`), `installUpdate(): Promise<boolean>`, `start(loadAdapter?)` (idempotent), `dispatch(BridgeEvent)`, `stop()`.
- **`BootStatus`** = `{ stage: BootStage, message: string|null, progressPct: number|null, localUrl, tunnelUrl, pin, updateAvailable: boolean, updateVersion: string|null }`. `BootStage = 'starting'|'running'|'no-tunnel'|'updating'|'error'`. `progressPct` is 0–100 and **non-null only while `stage==='updating'`** (reducer clears it otherwise); sticky url/pin survive updating events.
- **Adapter seam** `DesktopAdapter { invoke<T>(cmd): Promise<T>; listen(event, handler): Promise<()=>void> }`. Fake it in tests (see `desktop-bridge.test.ts` `fakeAdapter`): return canned invoke results + capture the `boot://status` handler to `emit()` events. `createDesktopBridge()` gives an isolated instance so runes state never leaks between tests.
- **Rust:** `BootStatus.progress_pct` serializes as `progressPct` (Option<u8>); `null` except during download. `publish()`/`boot://status`/`get_boot_status` unchanged.
