## Summary

Effects can now declare a **timebase**, and the generator bridge feeds voice-timebase effects
a **hit-relative clock** so they *start on the hit and restart on retrigger* instead of
unmasking a free-running animation mid-cycle.

- New registry field `EffectGenerator.timebase: 'voice' | 'absolute'` (default `'absolute'`).
  Interface, not convention — both render bridges (and, in S27, the thumbnail renderer) read it.
- The core bridge, for a `'voice'` generator, swaps the clock the generator reads:
  `ctx.timeMs = trig.ageMs` and a bridge-owned voice-local `ctx.transport` whose `beat` is
  derived from `age × bpm`. **Zero generator-signature change** — the effect keeps reading
  `ctx.timeMs` / `ctx.transport.beat`; the bridge just hands it a different clock.
- `'absolute'` generators (every effect except chase, for now) are untouched: the bridge feeds
  the engine wall-clock + transport exactly as before → byte-for-byte parity.
- **chase** is the tracer: a one-line `timebase: 'voice'` and **no body change** (its
  `ctx.transport.beat` read is now voice-local → beat 0 on the hit → step 0 → hoop 0).
- Mono voice steal already resets birth time (`spawn` stamps `bornAtMs = deps.timeMs` on every
  spawn) → a fast retrigger = a new voice whose age is 0. Locked with a test.
- Web offline bridge mirrors the core bridge identically (same flag, same `age×bpm` formula) →
  sim/engine parity.

## Context pack — the conversion recipe (READ THIS, S26 & S27)

### The seam (where the clock is swapped)

**Core:** `packages/core/src/voice/generator-bridge.ts`, `renderVoice`, the branch at
**:115**:

```ts
if ((gen.timebase ?? 'absolute') === 'voice') {
  const ft = frameTransport;
  const beats = (genTrigger.ageMs / 60000) * ft.bpm; // age×bpm — matches transport.beat accumulation
  voiceTransport.timeMs = genTrigger.ageMs;
  voiceTransport.beat = beats;
  voiceTransport.bar = Math.floor(beats / ft.beatsPerBar);
  voiceTransport.beatInBar = beats - voiceTransport.bar * ft.beatsPerBar;
  /* bpm / beatsPerBar / playing copied from the frame transport */
  genCtx.timeMs = genTrigger.ageMs;   // ctx.timeMs is now hit-relative
  genCtx.transport = voiceTransport;  // ctx.transport.beat is now voice-local
} else {
  genCtx.timeMs = timeMs;             // absolute: engine wall-clock, unchanged
  genCtx.transport = frameTransport;
}
```

`voiceTransport` is a single bridge-owned object refilled per voice — the shared frame
transport (the engine's, held by reference) is **never mutated**. `dt` is left as the real
frame delta in both cases (accumulators run in real time; only the phase clock resets).

**Web mirror:** `apps/web/src/lib/trigger-lab/render.ts`, `renderGeneratorVoice`, **:221**
(`voiceClock`), same `(ageMs/60000)*bpm` formula. Keep these two in lock-step — any change to
the voice-local clock must land in both, or sim/engine parity breaks.

### To convert an effect to restart-on-trigger (the S26 batch, ~20 effects)

1. **Declare it.** Add `timebase: 'voice',` to the generator object (see `chase.ts:16`). That is
   *usually the entire change*:
   - Effects that read **`ctx.timeMs`** directly (starfield, collisions, sacred-hogs,
     gravity-wells, comet-trails, orbit-rings, temp-sweep, most textures) now receive
     `age` there automatically → they restart with **no body edit**.
   - Effects that read **`ctx.transport.beat`** (chase, synced-hoops, strobe if beat-indexed)
     now receive the voice-local beat automatically → also no body edit.
2. **Audit stateful ones** (any generator with `createState`). Per-voice `genState` is already
   reset on (re)spawn — `voice-pool.ts:110` sets `slot.genState = null`, and `:119` sets
   `slot.bornAtMs = deps.timeMs` — so a retrigger gets a fresh state buffer *and* a fresh clock.
   The thing to verify per effect: its `createState`/seeding is deterministic and doesn't key
   off absolute time. comet-trails accumulates on `ctx.dt` (real frame dt, unchanged by
   timebase) into per-voice state → fine once `genState` reset is confirmed (it is).
3. **Leave base/ambient absolute.** breathing-kit, hue-rotate-kit, solid-base (and
   plasma-family textures *when used as looks*) must stay `'absolute'` — they must not
   phase-snap on section recall. Do **not** add the flag to them.
4. **Golden per converted effect** — template below.
5. Deliver the **41-effect timebase audit table** (which effects are voice vs absolute, and for
   voice ones whether they needed a body edit + the stateful-reset note). Mirror the S19/S22
   colour-audit format.

### Golden-test template (engine-level, `compositor.test.ts`)

The S25 block (`describe('Compositor — voice timebase / restart-on-trigger (S25)')`) is the
reusable harness. Copy it per effect:

- `chaseModel()` — a single 8-hoop drum so the active hoop is *distinct* per age (the 2-hoop
  `testModel()` aliases). For non-hoop-indexed effects, `testModel()` + a full-frame byte
  compare is usually enough.
- `chaseShow(polyphony)` — `genEffect('fx', <genId>, { attackMs: 0 })` so **age maps straight to
  animation** (age 0 → level 1 this same tick; the engine ticks spawn → `advanceEnvelopes` →
  composite, so attack 0 gives full level at age 0).
- `chaseAtAge(ageMs, polyphony)` — fire at t=0, `tick(0,0)` to spawn (born 0), `tick(ageMs,…)`
  to reach the age. Sample `e.frame()`.
- `chaseAfterRetrigger(ageMs)` — fire, run to t=500, retrigger on a **mono** bus, sample the new
  voice at `ageMs`. For `ageMs ≥ the release ramp (≥100ms)` the stolen voice is fully faded +
  reaped, so the frame is *only* the retriggered voice → compare byte-for-byte to
  `chaseAtAge(ageMs, 'mono')` to prove restart. (Age 0 overlaps the fading old voice; prove
  "starts at position 0" via a fresh `chaseAtAge(0)` instead.)
- `litHoopSet(frame, model)` — sorted hoop indices lit; the cheap assertion for hoop-indexed
  effects.

Assertions that pinned chase: `litHoopSet(chaseAtAge(0)) == [0]`; ages 0/200/800 →
`[0]/[1]/[6]` (advances, not frozen); `run() == run()` (deterministic); retrigger frame ==
fresh frame at ages 200 & 800 (restart). Plus an **absolute-parity lock**: plasma sampled at a
fixed engine time is identical regardless of birth time (age-independent) — reuse this shape to
prove any effect you *keep* absolute is unaffected.

Mono-steal age reset is unit-tested directly in `voice-pool.test.ts` (spawn A at t=0, spawn on
same mono bus at t=1000 → B.bornAtMs === 1000, A released).

### For S27 (timebase-aware thumbnails)

The `timebase` flag is your switch. `effect-thumb-render.ts` currently pins `ageMs = 0` forever
(§B root cause) — drive the synthetic trigger with a **looping age** (`age = now % 1600`) for
`timebase:'voice'` effects (so they fire-decay-repeat and show their real onset) and keep
**wall-clock** for `'absolute'` effects. Read `gen.timebase` off the registry exactly as the two
bridges do (`(gen.timebase ?? 'absolute') === 'voice'`). Reduced-motion static frame stays at
the representative ~400ms sample. After A (this slice) lands, most hit-relative thumbs fix
themselves once fed a non-zero looping age.

## Deviations

- **None from scope.** One design call worth surfacing: `timebase` is **optional with an
  `'absolute'` default** rather than a required field on all 41 effects. This keeps the 40
  unconverted effects byte-identical (no mass edit that would collide with S26's audit) and lets
  S26 flip effects one declaration at a time. The field is still "interface, not convention" —
  both bridges read `gen.timebase` explicitly. S26 will add the flag to the ~20 free-running
  effects and can decide whether to make it required once every effect declares one.
- **Mono-steal `bornAtMs` needed no code change** — `spawn` already stamped it on every spawn.
  The slice brief said "ensure mono steal resets `bornAtMs`"; verified it does and locked it with
  a regression test rather than editing working code.
