## Summary

The ADSR envelope editor was reworked onto the S23 v2 shape model and its single Curve slider
removed:

1. **Attack level is draggable.** The Attack handle — hardcoded to the top (`cy={yOf(1)}`) in v1
   — now moves in Y to set `attackLevel` (the peak the attack rises to, which is also the decay's
   start), and in X for the attack time. Readout gains a **P** (peak) column.
2. **Per-segment easing.** Each segment (attack / decay / release) carries its own `EaseSpec`.
   Click a segment band on the curve (or the keyboard-accessible **Segment** SegmentedControl) to
   select it, then the **EasePicker** (family Select + In/Out/In·Out) writes that segment's ease.
   The single Curve slider is gone.
3. **New reusable primitive** `lib/ui/EasePicker.svelte` — the family-grouped Resolume set
   (direction disabled for Linear). Added to the design system (`SectionPrimitives`) and
   `docs/design-system.html` regenerated in the same commit.
4. **Accessibility + motion.** The three SVG handles are `role="slider"`, focusable, arrow-key
   nudgeable (X/Y; Enter/Space selects the segment), with descriptive `aria-valuetext`.
   Reduced-motion disables the control transitions.
5. **Pure geometry seam** `trigger-lab/envelope-editor-geom.ts` — all handle↔shape math (coord
   mapping, handle anchors, segment bands/classification, drag→patch resolvers) lives here,
   DOM-free and unit-tested; the `.svelte` file is a thin view over it.
6. **Two S23-deferred follow-ups landed:**
   - `defaultAdsr()` is now **v2-canonical** (attackLevel 1 + linear eases, no `curve`) — new
     envelopes start eased; renders byte-identically to the pre-v2 default.
   - `migrateAdsr` is **wired on hydrate**: `normalizeGraphs` runs `migrateGraphsEnvelopes`
     beside `foldVelocitySwitches`, so persisted legacy shapes normalize to v2 on load
     (behaviour-preserving, idempotent, alias-stable).

## Context pack (for S34 — Envelope node inspector reuses this editor)

The S34 Envelope node inspector should **compose the same seams**, not re-derive them:

### The reusable ease control
`apps/web/src/lib/ui/EasePicker.svelte` — pure props `{ value: EaseSpec; onChange; ariaLabel?;
disabled?; class? }`. Family Select + direction SegmentedControl, direction auto-disabled for
`linear`. Exports `EASE_FAMILY_OPTIONS`, `EASE_DIR_OPTIONS` (single-sourced labels) and re-exports
`EaseSpec`. Demoed in the styleguide (`SectionPrimitives` → "Ease picker"). **Drop it straight
into the node inspector's per-segment rows.**

### The pure geometry seam
`apps/web/src/lib/trigger-lab/envelope-editor-geom.ts` — DOM-free, `import`-clean (tests need no
jsdom). Surface:
- `GEO {W,H,PAD}`, `xOf(t)`, `yOf(v)`, `toUnit(clientX, clientY, rect): {t,v}` — coordinate mapping.
- `handleAnchors(a)` → `{ attack, sustain, release }` unit anchors (attack's `v` carries
  `attackLevel`).
- `segmentBands(a)` → per-`Stage` `[t0,t1]` clickable bands; `segmentAt(a, t)` → `Stage | null`
  (null on the flat sustain plateau).
- `dragAttack/dragSustain/dragRelease(a, t[, v]) → Partial<AdsrShape>` — the drag→patch resolvers
  (attack X+Y→attackLevel, sustain X+Y, release X-only), with the ordering clamps baked in.
- `NUDGE` (keyboard step), `type Stage = 'attack'|'decay'|'release'`.

If S34 wants a canvas-embedded editor rather than a modal, build its view on these + EasePicker —
they carry all the shape logic.

### The store commit pattern (what the current editor does)
The editor never mirrors state: each edit computes a fresh shape and calls
`store.setEnvAdsr(node, key, { ...adsr, ...patch })`, which persists the `AdsrShape` and
regenerates `points` via `adsrToPoints` (single source). `store.getEnvelope(node, key)?.adsr`
re-derives reactively. S34's inspector will use its own store seam (per doc 10's node model) but
the same "compute fresh shape → persist → re-derive" discipline applies.

### Deliberately NOT extracted
The store-bound Dialog wrapper (`EnvelopeEditor.svelte`) was **not** split into a prop-driven
"view" component — that would have been a bigger refactor than S24's scope. The genuinely reusable
parts (EasePicker + geometry) are already standalone. If S34 finds it wants the whole editor body
as a prop-driven view, extracting `EnvelopeEditorView.svelte` from the current markup is a clean,
mechanical follow-up (the geometry + ease logic already live outside the component).

### Segment vs handle vocabulary (gotcha)
Two vocabularies coexist: **handles** are `attack | sustain | release` (the middle "D·S" handle),
**segments/eases** are `attack | decay | release` (the middle is "decay"). The editor maps the
sustain handle → the decay segment on grab/select. `envelope-editor-geom.Stage` is the *segment*
vocabulary; the drag resolvers are named by *handle*.

## Deviations

- **Non-modal "view" component not extracted** (see Context pack → "Deliberately NOT extracted").
  Kept S24 in scope; the reusable logic (EasePicker + geometry) is standalone, so S34 loses
  nothing.
- **Legacy non-representable-`curve` shapes show a Linear ease in the selector** until a segment is
  explicitly edited. Such shapes can only exist from pre-S24 persistence (the curve slider is gone
  and `defaultAdsr` is v2); hydrate migration retains their `curve` fallback so the *rendered*
  curve stays exact, but the discrete ease set can't represent an arbitrary power law, so the
  selector defaults to Linear. Editing a segment writes an explicit ease (exact thereafter). This
  is the honest reading of S23's "curve retained on the fallback".
- **`derived_inert` warnings in the EnvelopeEditor jsdom test** — benign Svelte 5 +
  `@testing-library/svelte` teardown noise (a component `$derived` read as the effect is
  destroyed on unmount). Warnings only; all 6 tests pass. Any store-bound component test exhibits
  this; no behaviour impact.
- **Handle hit radius kept at 16px (32px)** rather than raised to the 40px min-hit-area guideline —
  the three ADSR nodes sit close together in a dense editor, and 40px hit circles would overlap
  (which the guideline also forbids). 32px is the reasonable compromise for this specialized
  control.
