# S17 — LayersDock server truth (E, issue #50)

## Summary
The Layers/Buses dock now shows the voices the **server engine is actually sounding** when the link
is open, instead of the offline sim's stale list. This closes the last dock↔visualiser divergence:
with S15 spawning section looks in the engine, a connected recall's looks now appear in the dock's
per-voice chips **and** the visualiser frame — they can no longer disagree.

- **Wire**: the `stats` voice payload gains per-voice detail (`VoiceStat[]`) — id / bus / effect /
  mode / combined level / hue / releasing / via. Built only in `engine.stats()` at the ~2 Hz
  telemetry cadence, never on the render hot path.
- **Source-selection** is a pure module (`dock-voices.ts` → `selectDockVoices`): connected ⇒ server
  voices, offline ⇒ sim voices. The dock renders one normalized `DockVoice` and never knows which
  side it came from.
- **Bus meters** follow the same authority rule: `snapshot()` no longer clobbers the server-streamed
  `busLevels` every rAF frame while connected (it was overwriting the ~2 Hz server value ~30×/s).

Rebased onto the current group/E (S14 + S15 landed after this slice branched) — S15 is the other
half of the divergence fix, so S17 sits directly on top of it and the acceptance is real end-to-end.

## Context pack (READ THIS — S16 / future slices)

**The dock's source-of-truth rule.** `store.dockVoices` (`store.svelte.ts`, a `$derived`) is the ONLY
thing the dock reads for per-voice chips. It calls the pure `selectDockVoices({ connected, simVoices,
serverVoices })`:
- `connected` is **`store.link === 'open'`** — the same firing/authority gate the whole E-family uses
  (doc 03 / S12). NOT `useServer` (the stricter visualiser-frame gate): the dock owns no pixels, so it
  adopts server voices the instant the link opens, without waiting for a frame. Match this gate for any
  new "which source" decision in the dock.
- `simVoices` = `store.voices` (the sim snapshot); `serverVoices` = `store.serverVoices` (set from the
  `onStats` voice payload, cleared on link drop).

**The `DockVoice` view model** (`dock-voices.ts`) is deliberately minimal — only what a chip draws
(id, busId, effectId, mode, level, hue, releasing, via). `level` is pre-folded `level*deckGain`; `hue`
is resolved (0 when the effect exposes none). If a chip needs a new field, add it to `VoiceStat` (wire),
populate it in `engine.stats()`, and map it in BOTH `simVoiceToDockVoice` and `serverVoiceToDockVoice`.

**Bus levels authority.** `busLevels` is now server-owned while connected: `onStats` writes it, and
`snapshot()` only writes sim levels when `link !== 'open'`. Don't reintroduce an unconditional
`busLevels` write in the rAF loop or you'll re-clobber the server value.

**Where per-voice detail comes from.** `engine.stats().voices` walks the active voice pool once per
stats call (~2 Hz) — cheap, off the hot path. `voices.length === voiceCount` by construction. The host
passes it straight through (`main.ts` statsTimer → `voice: { …, voices }`); core `VoiceStat` and
protocol `VoiceStat` are structurally identical, so no mapping layer sits between them.

**effectName still resolves connected.** The dock labels chips via `store.sim.effectName(v.effectId)`
— the effects registry is shared/adopted from server `state`, so a server voice's `effectId` resolves
to a name. (If an unknown id ever slips through, `effectName` falls back to the id — no crash.)

## Deviations
- **`ui-light` — no new design-system component.** The dock is a pure data-source swap: identical
  markup/styles, no new reusable primitive, so no styleguide entry and no `pnpm design-system`
  regeneration. Applied the `/make-interfaces-feel-better` lens to the touched surface — the pill chips
  already use concentric radius, a subtle `releasing` opacity (softer exit state), and the meter has a
  `prefers-reduced-motion` guard; no `transition: all`. Deliberately did **not** bolt JS enter/exit
  transitions onto the chips: they'd behave differently offline (60 fps) vs connected (~2 Hz telemetry)
  and would need separate reduced-motion handling — gratuitous for a source-selection slice.
- **Bus-level clobber fix included (in scope).** Beyond the per-voice list, `snapshot()` was overwriting
  the server's streamed `busLevels` every rAF frame. Fixing it is squarely "the dock derives from
  server-streamed voice/**bus** state" (doc 04 §3, S17 acceptance "dock and visualiser can no longer
  disagree"), so the meter is now server-truth when connected too. Covered by `store.dock-source.test.ts`.
- **Rebased onto current group/E** (S14 + S15 merged in after branch). One trivial conflict (the
  `engine.ts` `./types` import block — kept both S15's `ParamValues` and this slice's `PlayMode`); all
  other commits replayed clean. Full sweep re-run green on the rebased branch.
