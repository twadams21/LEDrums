# S42 — Library UI + naming pass (group J, #53) — implementer report

Branch `slice/S42` off `group/J` (S41 tip). Library surfaces + naming + the engine/UI consumption of
S41's resolved view, so a referenced song is navigable, playable, and editable — not just listed.

## Summary
- **Songs source split**: Objects → Songs detail has two groups — **This show** (local `SongRow`s +
  resolved library refs as `LibraryRefRow`) and **Song Library** (`LibrarySongRow` pool).
- **Row actions**: local → *Add to library* (`exportSongToLibrary`); pool → *Import to show* ↔
  *Remove from show* by state; ref → *Detach copy* (`detachSongReference`) / *Remove from show*
  (`removeSongReference`, new — un-import without cloning). Rename on a ref/pool row edits the canonical copy.
- **Used-by + delete guard**: pool rows show "Used by N shows"; Delete disabled while referenced, the verb
  naming the block reason (mirrors the store guard via pure `librarySongRows.deletable`).
- **Resolved-view consumption (S42 engine/UI wiring)**: `activeSong`/`setActiveSong`, `selectedGraph`,
  hit-firing, `graphLabel`/`triggerSource`, `selectGraphInSection`, the `SongRail` (now the resolved
  **Setlist**), and the Inspector recall all read the resolved view; `buildShow` (connect + `syncShowToServer`)
  sends a RESOLVED ShowSource so the engine `recallSection`s a referenced section. Editing a referenced graph
  writes through S41 aliasing to the library copy (propagation); `toAuthored` stays refs-not-copies.
- **Naming**: pool = "Song Library", setlist rail = "Setlist", show group = "This show". No
  "Show Song Setlist manager" remnants existed (grep-verified).
- **core**: `assertShowIntegrity` exempts `lib:<id>/…` keys (routed by trigger source, not the namespaced key).
