## What changed
- **Closure extraction** (`store/song-library.ts`): pure `extractSongClosure(song, sources, librarySongId)`
  → `LibrarySong` (self-contained namespaced closure). "Reaches" = referenced graphs present in sources;
  play-node `effectId`/`presetId` + section-look effects; look presets = `<effectId>:default`. Modifier/
  lfo/cc/env nodes pull nothing; `modifierId`/`generatorId` are global (not namespaced).
- **Persistence doc** (`persistence.ts`): `SongLibrary`/`LibrarySong`, `SONGS_STORAGE_KEY` `ledrums:songs:v1`,
  `SONGS_VERSION`, `serialize/deserialize/loadSongLibrary` + `coerceLibrarySong` — versioned, defensive (never throws).
- **Server seam** (`named-blob-store.ts`): `createNamedBlobStore(file)` factory; `show-library.ts` rewritten as a
  thin wrapper (exports/behaviour unchanged); new `song-library.ts` (`default.songs.local.json`).
- **Protocol + server wiring**: `setSongLibrary` client msg, `songLibrary` server msg + `state` field; main.ts
  load/autosave/state/adopt/relay + boot flush + startup diagnostic — all mirror the show library.
- **Web adopt** (`store/song-library-sync.ts`): `SongLibrarySync` (cold-load-adopt/seed/follow/push), a sibling
  of `ShowLibrarySync`. Store rune wiring deferred to S41 (owns refs/content).
