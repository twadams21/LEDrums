# S18 ‚Äî Enum params end-to-end (group F, PRD #45 / stories 22‚Äì25)

## Summary

Enum params (radial-wash `mode`, wipe-3d `axis`/`mode`) were stuck at the generator's own
default and uneditable. Two things blocked them: `ParamValue` was `number | boolean` (a string
enum value couldn't be stored/typed), and `mapParamSpec` returned `null` for enum/color specs
(so the value never even reached a play node). The **engine path already handled strings** ‚Äî the
generator bridge overlays every live param key onto the generator defaults ‚Äî so this slice only
widens the authoring seam:

- `ParamValue` gains `string`; `ParamSpec.kind` gains `'enum' | 'color'` + `options` ‚Äî in both
  the core voice types and the web sim mirror (kept structurally identical so a web `EffectDef`
  still flows into a `voice.Show` on `setShow` with no conversion).
- `mapParamSpec` is now **total** over all four `ParamType`s (no silent drop): enum ‚Üí a Select
  control, color ‚Üí a colour spec. Exported for testing.
- `PlayNodeInspector` renders enum params as a `Select`. (Color‚Äôs inspector control ‚Äî the
  write-through swatch ‚Äî is S19‚Äôs; no effect declares a `type:'color'` param yet.)

Then integrated the current `group/F` (which had gained S19‚Äôs ColorSwatch + colour pass): one
trivial conflict in the inspector `<script>` (kept both helper sets). Enum Select and S19‚Äôs
ColorSwatch coexist; full sweep green on the integrated tree.

## Context pack

### The widened `ParamValue` seam (for S31 ‚Äî modulation / builds on enum params)

- **Two mirrored definitions, must stay identical**: `packages/core/src/voice/types.ts:76`
  (`ParamValue = number | boolean | string`) and `apps/web/src/lib/trigger-lab/sim.envelopes.ts:11`.
  Widen BOTH together or `voice.Show` (setShow payload) type-breaks ‚Äî a web `EffectDef`/play-node
  is assigned into a `voice.Show` without conversion, relying on structural identity.
- **`ParamSpec` (both files)** gained `kind: '‚Ä¶' | 'enum' | 'color'` and `options?: string[]`.
- **The engine already routes strings unchanged** (no S18 change here ‚Äî pointers for whoever
  builds on it):
  - `packages/core/src/voice/compositor.ts:58` `applyEffectiveParams` copies *every* `v.params`
    key into `liveParams`; the envelope loop skips non-number specs at the
    `spec.kind !== 'number'` guard (line 68) ‚Äî so strings pass through, never enveloped.
  - `packages/core/src/voice/generator-bridge.ts:80-85` overlays *all* live keys onto the
    generator defaults (`defaultParams(gen.paramSpec)`), then `gen.render` reads them via
    `pstr` (`packages/core/src/effects/types.ts:49`).
  - `packages/core/src/voice/eval-graph.ts:286` `resolveNodeParams` returns `node.params`
    verbatim (linked ‚Üí the shared preset‚Äôs params).
- **Leaf `num()` helpers were widened** to `number | boolean | string | undefined` at
  `voice/compositor.ts:34`, `voice/pattern-renderer.ts:18`, `trigger-lab/render.ts:21`,
  `EffectThumb.svelte` (~:33). `node-options.ts:108` `num` / `:114` `fmt` already took
  `ParamValue | undefined` (no change). If you widen `ParamValue` again, sweep these call sites.

### The spec ‚Üí control mapping seam

- `apps/web/src/lib/trigger-lab/fixtures.ts` `mapParamSpec(spec: CoreParamSpec): ParamSpec` is
  now **total and exported** ‚Äî never returns null. enum ‚Üí `{kind:'enum', options, default:string}`
  (default falls back to `options[0]` if the core default isn‚Äôt a string); color ‚Üí
  `{kind:'color', default:'#rrggbb'}`. `GENERATOR_EFFECTS` no longer `.filter(p => p !== null)`.
- `PlayNodeInspector.svelte` param loop: `number ‚Üí Slider`, `enum ‚Üí Select` (labels via
  `titleCase`, value via `str`), `else ‚Üí Toggle`. **`color` deliberately has no branch here** ‚Äî
  it would fall to the Toggle `{:else}`, but no effect declares one; a future real `type:'color'`
  param needs an explicit `{:else if spec.kind === 'color'}`. S19‚Äôs colour swatch is a *different*
  mechanism (`hasColorSwatch`, keyed on numeric hue+saturation+brightness), not a `type:'color'`
  control.

### For S20 ‚Äî colour batch 2 (shares radial-wash + wipe-3d with this slice)

- **Do NOT re-widen `ParamValue`** ‚Äî S18 did it. S20 only adds numeric `saturation`
  (+brightness) params and swaps the hardcoded saturation, exactly like S19 did for hit/trigger
  effects.
- `packages/core/src/effects/impl/radial-wash.ts`: the saturation seam is line 64
  `hsvToRgb(hue, 1, bri * intensity)` ‚Üí `hsvToRgb(hue, sat, ‚Ä¶)` + a `saturation` ParamSpec. The
  `mode` enum spec is line 36; `waveRadius()` (lines 11-23) has **no default switch arm** (an
  out-of-domain mode ‚Üí `undefined` radius ‚Üí NaN ‚Üí nothing lit) ‚Äî safe, but don‚Äôt assume a valid
  mode elsewhere. My enum change did **not** touch the colour math.
