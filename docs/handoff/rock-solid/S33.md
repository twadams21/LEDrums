## What changed
- `voice/modulation.ts`: the mapping model — `Mapping` (targetParam/source/amount/invert/
  rangeMin/rangeMax), `ModSource` (envelope arm; `lfo`/`cc` extension seam), and the
  base-relative **summed-and-clamped** sweep `applyModulations` + `envelopeToMapping`.
- `applyEffectiveParams` (core) / `effectiveParams` (web sim) consume `Voice.modulations`;
  `applyModifierChain` gained an optional `modCtx` and now samples `ResolvedModifier.modulations`
  onto each link per-frame (both compositor paths + both web-sim paths). **One model, two carriers.**
- Spawn/pool + `PlayAction` (core & sim) carry `modulations` for S34; `modifiers/types.ts`
  `ResolvedModifier.modulations` replaces the reserved `env` note.
- `voice/modulation-parity.ts`: reusable legacy-env ↔ mapping parity fixture (cases + phases +
  `legacyEnvValue`/`mappingEnvValue`). Tests: 15 core + 2 web.

## Deviations
- Doc 10's contribution prose ("scale into [min,max] weighted by amount") is realized
  **base-relative** (`amount·(scaledTarget − base)`) — the only form that reproduces the legacy
  env sweep. Parity holds to **floating-point precision** (12 digits), not bit-identity: the new
  model additionally range-clamps, a no-op within range but a ~1 ULP snap at an exact endpoint.
- Modifier-param modulation: S33 lands the **application seam** (field + runner + tests inject
  mappings). Populating it from the graph (incl. legacy modifier `node.env`) is S34 — kept out of
  `resolveModifierChain` to respect the "no graph work" scope boundary.
