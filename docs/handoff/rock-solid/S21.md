# S21 ‚Äî Colour batch 3: textures (Group F, issue #51)

## Summary

The thirteen **texture** effects can now make white and pastels, not just fully-saturated
hues. Every texture gained a `saturation` param (default 1) threaded into every `hsvToRgb`
call, so `saturation` 0 washes a lit texture to white while defaults reproduce the old look
byte-for-byte. Eleven read as a single base colour (a churning/spread palette around one hue)
and therefore light up the write-through **ColorSwatch** automatically ‚Äî `lava-lamp` and
`caustics` even gained a base `hue` param they previously hardcoded, so the swatch can recolour
the lava/water. The two genuinely **multi-colour** textures ‚Äî `tunnel` and `rainbow-flow`, which
sweep a *full* hue wheel across space ‚Äî instead expose a hue **offset/range** scheme and
deliberately carry no bare `hue` param, so they skip the swatch (a picker can't represent a
rainbow). This applies the S19 colour-pass recipe verbatim; no redesign.

## Context pack (brief) ‚Äî single vs *multi* judgment (for S22)

The whole batch is the same shallow change: **add `saturation` (default 1) and multiply it into
whatever the effect already used in the S-slot** ‚Äî a literal (`1`, `0.95`, `0.9`) or a computed
sat (fire/perlin/lava/caustics' heat- or density-driven wash). Default 1 preserves output; sat 0
gives achromatic white. When an effect hardcoded its palette hue with no `hue` param
(`lava-lamp` 8¬∞, `caustics` 200¬∞), add a base `hue` too so the swatch can drive it.

The one real judgment call is **single (swatch ‚úì) vs *multi* (offset/range, no swatch):**

- The swatch is auto-applied iff params include **all three** of `hue`+`saturation`+`brightness`
  (`COLOR_KEYS` in `PlayNodeInspector.svelte`). To skip it, an effect must not carry a bare
  `hue` ‚Äî name the base `hueOffset` and add a range param.
- **Only a *full-wheel spatial sweep* is *multi*.** `tunnel` (angle ‚Üí ¬±180¬∞) and `rainbow-flow`
  (`(u+v)` ‚Üí 360¬∞) show every hue at once, so a single picker is misleading ‚Üí `hueOffset`
  (+`hueRange` on tunnel), no swatch.
- **"Spreads hue" is *not* *multi*.** An effect that merely spreads hue around a base
  (`plasma` `hueSpread` 120¬∞, `ripple-pond` 80¬∞, `spiral`/`interference` ‚â§ ~88¬∞) stays
  single-hue **with a swatch on the base** ‚Äî matching S19's `synced-hoops` precedent
  (`hue`+`hueSpread` ‚Üí swatch ‚úì). Do not promote these to *multi*.
