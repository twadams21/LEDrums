# S16 — Looks authoring UI + model (E, issue #50)

## Summary
Section "looks" are now **authored, persisted, and editable** — the last piece that makes the
S15 engine parity useful. A section's per-bus loop effect (its "look") moved from fixture-only
data onto the authored `SetlistSection` model, gained a persistence migration, and got a real
authoring surface: the Section inspector's new **Looks** group (one effect picker per bus, with
a None option). Picking a look persists via autosave and live-resyncs to the engine, so
**pick a look → recall the section → it plays on the engine and the visualiser**.

The core insight: rather than keep the fixture `store.sections` look-array in sync with the
authored setlist (two sources of truth), **`store.sections` became a `$derived` projection of
the active song's authored sections**. Authored looks are now the single source, flowing
unchanged through the existing S15 bridge point (`show-builder.ts:54`) to `Show.sections` — no
`show-builder` change was needed. The fixture demo looks were moved into `seedSongs`, so they're
now editable authored content.

- **Model** (`setlist.ts`): `SetlistSection` gains `looks: Record<busId, effectId|null>`
  (default `{}`); `makeSection`/`cloneSection` carry+copy it; new pure `setLook` op (immutable,
  idempotent).
- **Persistence** (`persistence.ts`): `migrateSongs` coerces `looks` defensively + defaults to
  `{}` — a pre-S16 show loads with empty looks (== no looks == unchanged), idempotent on re-run.
- **Bridge** (`store.svelte.ts`): `store.sections` derives from the active song's authored
  sections, so `buildShow` (unchanged) and the offline `setActiveSection` recall both read
  authored looks. `setLook` store mutation persists + live-resyncs; offline it re-morphs the
  active section immediately.
- **UI** (`SectionInspector.svelte`): a Looks group — per-bus rows (bus icon + Select), options
  = None + that bus's home effects. Composed from design-system primitives; artifact regenerated.

## Context pack (brief)

### Where looks live now (the data model)
- **Authored source of truth:** `SetlistSection.looks: Record<busId, effectId|null>` on
  `store.songs[].sections` (`setlist.ts`). `null`/absent = None. Persisted with `songs`;
  migrated by `migrateSongs` (pre-S16 → `{}`).
- **`store.sections` is now `$derived`** (`store.svelte.ts`, next to `activeSection`): projects
  the **active song's** sections → `Section[] { id, name, looks }`. This is what `buildShow`
  reads for `Show.sections` and what the offline `setActiveSection` recall resolves. There is no
  longer a separate fixture look-array — `seedLookSections` was removed; `seedSongs` seeds the
  fixture looks onto the authored song.
- The engine seam is **unchanged from S15**: `buildShow` (`show-builder.ts:54`) still does
  `source.sections.map(s => ({ ...s, looks: { ...s.looks } }))`; because `source.sections` now
  carries authored looks, the bridge "just works". `VoiceBusEngine.spawnSectionLooks` reads
  `Show.sections.find(id).looks` (S15) — no core change in S16.

### The authoring path (end to end)
`SectionInspector` Looks Select → `store.setLook(sectionId, busId, effectId|null)` →
`updateActiveSong(setlist.setLook(...))` mutates `songs` → the debounced autosave persists AND
`syncShowToServer` re-sends `setShow` + re-recalls the active section (connected), so the edited
active section re-morphs with the new look. Offline, `setLook` re-morphs the local sim inline
(an immediate connected recall would race the not-yet-sent Show, so it's gated to offline).

### Picker scoping (deliberate)
Each bus's Select lists **only effects whose home bus is that bus** (`e.busId === bus.id`) + None.
A look spawns on the effect's own home bus (S15's release/spawn asymmetry), so listing only
home-bus effects keeps the pick and the audible result on the same bus (no cross-bus surprise).

## Deviations
- **`store.sections` changed from `$state` (fixture seed) to `$derived` (active song).** The
  assignment/plan says "extend the S15 bridge point"; the cleanest extension was to make the
  bridge's *input* authored rather than adding a second mapping. This is the reconciliation the
  S15 handoff named ("reconcile the fixture `store.sections` with authored per-section looks").
  It also removes a latent bug (the old fixture array never followed a show switch) and needs no
  `show-builder` edit. `store.sections` was never mutated or persisted, so nothing else changed.
- **Fixture looks moved into `seedSongs`** (were `seedLookSections`, now removed). The demo
  looks (intro/verse/chorus) are now authored + editable + persisted, not a parallel fixture.
- **Offline-only inline re-morph in `setLook`.** Connected re-morph rides the existing debounced
  resync (setShow + re-recall) rather than an immediate recall, which would spawn the stale look
  before the new Show lands. Mirrors the S15 `setActiveSection` link gate.
- **No new reusable component / styleguide entry.** The Looks group is an inspector-local
  composition of existing primitives, so nothing was added to `lib/ui`; the artifact was still
  regenerated (its bundle embeds the updated components).
- **Did not touch the S17 surface.** S17 (LayersDock server truth) has since landed on group/E;
  it touches LayersDock/dock-voices/protocol/engine-stats — zero overlap with S16's files, so the
  slice/S16 → group/E merge is clean.
