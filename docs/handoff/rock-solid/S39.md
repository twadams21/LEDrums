## What changed
Presets stop being a live binding and become **snapshots**; every play node always owns its params.

- **Core** (`voice/types.ts`, `voice/eval-graph.ts`): drop `GraphNode.linked`; `resolveNodeParams`
  deleted — a play `PlayAction.params` is always `node.params`. `EvalState.presetsById` stays (still
  used by section-look defaults). Test node fixtures across core + server stripped of `linked`.
- **Web model** (`sim.ts`, `sim.graph-compilation.ts`, `fixtures.ts`): drop `PlayBlock.linked`,
  `makeNode`/`nodeFromBlock`/seed-fixture `linked`, and the linked branches in the sim's
  `resolveParams`/`resolveNodeParams`.
- **Migrator** (`store/hydrate.ts`): new `materializeLinkedNodes` / `…All` — for each
  `linked:true` play node, copy its shared preset's params onto the node (exactly the old
  `toggleLink` unlink branch; keeps node params when the preset is unknown) so the **effective
  render is unchanged**, then strip `linked` from every node. Idempotent + alias-stable (mirrors
  `foldVelocitySwitch`), wired into `normalizeGraphs` before the switch/env folds. `normalizeGraphs`
  gains a `presetParamsFor` arg (store passes `presetById(id)?.params`).
- **Store** (`store.svelte.ts`, `store/objects.ts`, `store/graphs.ts`): `liveParams`/`setParam`
  always node-local (no shared-preset write-through); `toggleLink` removed; `selectPreset` forks the
  preset onto the node; **new** `applyPreset` (reset params to the current preset) and
  `saveNodeAsPreset` (snapshot node params → new preset, point `presetId` at it). `presetUsageCount`
  is now advisory `presetId`-provenance (still gates delete).
- **UI** (`PlayNodeInspector.svelte`, `ClipSettings.svelte`, `node-options.ts`): drop the
  linked/instance SegmentedControl + `LINK_OPTS`; add **Apply** (RotateCcw) + **Save-as-preset**
  (BookmarkPlus) `IconButton`s beside the preset select (design-system primitives, no new
  component). Footer copy updated.

## Context pack (S40 — library closure extraction)
- Params are now always node-local — no live preset indirection to resolve across shows, exactly the
  simplification doc 07 §B ordering assumed. A song's closure = sections → graph keys → each play
  node's `effectId` + `presetId` (provenance) + `params`. `presetId` is a **label**, not a runtime
  dependency, so a closure that carries effects + presets by id is sufficient; a node whose preset is
  absent still renders (its own `params`).
- `presetUsageCount(graphs, id)` already counts `presetId` references across all graphs — reuse for
  the library's usage/delete guard.
