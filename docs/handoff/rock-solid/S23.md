## Summary

The ADSR envelope shape gained two capabilities and was collapsed to a single source:

1. **`attackLevel`** — the peak the attack rises to (default 1); it also becomes the
   decay's starting level. Absent ⇒ 1 ⇒ byte-identical to the old renderer.
2. **Per-segment easings** — each of attack/decay/release carries its own `EaseSpec`
   `{ fn, dir }` drawn from the Resolume-familiar set
   (`linear|quad|cubic|quart|expo|sine|circ|back|bounce|elastic` × `in|out|inOut`),
   implemented once in a new pure `easing.ts`. A segment with no `EaseSpec` falls back
   to the legacy single `curve` power law, so un-migrated shapes render unchanged.
3. **Single source** — the shape model, easing library, and sampling code live in
   `packages/core/src/voice/{types,easing,envelope}.ts`. The web sim
   (`sim.envelopes.ts`) now **re-exports them from `@ledrums/core` by value** — the
   web symbols are the *same references* as core's (the `computeDelayMs` precedent).
   The ~140 lines of duplicated shape/easing/sampling code in `sim.envelopes.ts` are
   gone; only the param-value primitives remain local (S18 owns their widening).
4. **`migrateAdsr`** — behaviour-preserving, idempotent migration from the legacy
   single-`curve` shape to the v2 per-segment form.

## Context pack (for S24 + S33)

### The shape type
`packages/core/src/voice/types.ts` → **`AdsrShape`**:
```ts
attack, decay, sustain, release: number
attackLevel?: number     // 0..1 attack peak / decay start (default 1)
curve?: number           // legacy -1..1 tension; drives a segment ONLY when its *Ease is absent
attackEase?, decayEase?, releaseEase?: EaseSpec
```
plus `EaseFn`, `EaseDir`, `EaseSpec` (`{ fn: EaseFn; dir: EaseDir }`). `curve` is now
optional — anything reading `a.curve` must default it (`a.curve ?? 0`).

### The easing registry
`packages/core/src/voice/easing.ts` → **`ease(spec: EaseSpec, t: number): number`**.
Internals: the `IN: Record<EaseFn, EaseImpl>` table holds each family's `easeIn` form;
`out`/`inOut` are derived (`out(t)=1−in(1−t)`, `inOut` mirrors at the midpoint).
Endpoints are **snapped exact** (`ease(_,0)===0`, `ease(_,1)===1`) for every family.
`back`/`elastic` intentionally overshoot outside 0..1 mid-range — callers clamp.
To add a family: add one `easeIn` entry to `IN`. Exported via `voice.ease` / the
`EaseFn` list for a family-grouped selector (S24's ease picker).

### The sampler
`packages/core/src/voice/envelope.ts`:
- **`adsrToPoints(a: AdsrShape, n = 48): EnvPoint[]`** — shape → breakpoints. Attack rises
  to `peak = attackLevel ?? 1` via `segEase(attackEase, curve, ·)`; decay goes `peak→sustain`;
  release goes `sustain→0`. `segEase(spec, curve, t)` = `spec ? ease(spec,t) : easeCurve(t,curve)`.
- **`sampleEnvelope(env, phase): number`** — unchanged; piecewise-linear over `env.points`.
- The compositor sampler `applyEffectiveParams` (`voice/compositor.ts:58`) is **shape-agnostic**
  (it sweeps envable params by `sampleEnvelope`), so richer shapes needed **no** compositor change.
  **S33 (modulation core)** should consume `sampleEnvelope` / `adsrToPoints` from here directly —
  `attackLevel` + eases ride along transparently, no S33 change required to benefit.

### The migrator
`packages/core/src/voice/envelope.ts` → **`migrateAdsr(a): AdsrShape`**. Idempotent.
`curve === 0` → explicit linear eases (curve dropped). Non-representable curves (e.g. any
0.05-step slider value) are **kept on the `curve` fallback** so migration never alters a
sampled point — see `curveToExactEase` (only `curve===0` is IEEE-754-exact; `Math.pow`-based
curves would risk last-bit drift against a repeated-multiply ease). A shape that already
carries any authored ease is returned unchanged (only `attackLevel` defaulted) — authored
v2 shapes are never clobbered.
> **Not yet wired on load.** `migrateAdsr` is exported but not called from hydrate. This is
> intentional and safe (the fallback already renders legacy shapes identically). When S24
> starts authoring eases, invoke it in `store/hydrate.ts` next to the `foldVelocitySwitch`
> precedent so persisted shapes normalize on load.

### The web seam (single source)
`apps/web/src/lib/trigger-lab/sim.envelopes.ts` re-exports (by value, so identity holds)
from `@ledrums/core`: `envShape, presetPoints, defaultEnvelope, cloneEnvelope, defaultAdsr,
adsrToPoints, sampleEnvelope, migrateAdsr, ease` + types `EnvKind, EnvPoint, Envelope, EnvMap,
AdsrShape, EaseFn, EaseDir, EaseSpec`. Kept **local** (S18 territory): `ParamValue, ParamValues,
ParamSpec, clampUnit, ENV_KINDS`. `./sim` (sim.ts) now also surfaces `migrateAdsr, ease, EaseSpec,
EaseFn, EaseDir`.

## Deviations

- **`sim.envelopes.ts` kept as a thin re-export shim rather than deleted outright.** Deleting the
  file would force relocating the param-value primitives it still owns, colliding with S18
  (ParamValue widening). "Delete the duplicates" is satisfied by removing the duplicated *code*
  (shape/easing/sampling now imported, not copied); the shim keeps the `./sim.envelopes` surface
  stable for its three importers.
- **Migrator promotes only `curve === 0` to eases; other curves ride the retained `curve` fallback.**
  A discrete easing set cannot reproduce an arbitrary power law bit-for-bit, and the editor's curve
  slider emits 0.05 steps (mostly non-cardinal). Keeping the fallback makes parity *exact for every
  curve* rather than approximate — the honest reading of "sample-identical". Rationale is in the
  `curveToExactEase` doc-comment.
- **`migrateAdsr` is not yet invoked on hydrate** (see context pack) — safe no-op today; wire in S24.
- **One-line UI compat touch** (`EnvelopeEditor` curve slider `?? 0`) to satisfy the now-optional
  `curve` type. Behaviour-preserving and non-visual, so no design-system pass — the editor's real
  rework (and its `/make-interfaces-feel-better` pass) is S24.
