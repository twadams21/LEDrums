## Summary

The thumbnail renderer now drives its synthetic trigger with a **looping hit age** and reads
each generator's declared **timebase**, so hit-relative effects visibly *fire → decay → repeat*
instead of freezing at their age-0 frame.

- **Looping age.** The synthetic trigger's `ageMs` sawtooths over `THUMB_LOOP_MS` (1600ms —
  3.2 beats @120bpm): `ageMs = ((tMs % LOOP) + LOOP) % LOOP`. Effects that decay off
  `trig.ageMs` (whole-drum, burst, radial-wash, whole-kit, follow-hoop, lightning…) now fire on
  the hit and decay, then re-fire each loop. Previously `ageMs` was pinned to 0 forever → a
  frozen full-brightness frame (§B root cause).
- **Timebase-aware clock.** The renderer reads `(gen.timebase ?? 'absolute') === 'voice'`
  exactly as the two render bridges do, and swaps the clock the generator's `ctx` reads:
  `'voice'` → the hit-relative looping age (animate from onset, restart each loop);
  `'absolute'` → the free-running wall-clock `tMs` (base/ambient loops keep phasing). The
  transport `beat` is advanced from that same clock (`beat = clock × bpm / 60000`, the bridges'
  formula) so beat-indexed voice effects (chase) step instead of freezing at beat 0.
- **Written against the flag, not effect ids.** No effect is special-cased, so S26's
  timebase flips (sibling slice, ~20 effects) light up in the thumbnails automatically — an
  effect flipped to `'voice'` starts loop-restarting in its thumb with zero thumbnail change.
- **Drum-key fix.** The synthetic trigger's `drumId` was `''`, which matches no drum in the
  thumb model (its single drum is `'thumb'`), so drum-keyed effects (whole-drum, burst,
  radial-wash) were rendering a **black** frame — not the "frozen full brightness" §B assumed.
  Set `drumId: 'thumb'` so they find their drum and light. Regression-locked.
- **Reduced motion unchanged.** The component still samples one static frame at the
  representative ~400ms (`EffectThumb.svelte` reduced-motion branch untouched); at that `tMs`
  the loop age is 400ms → a representative mid-fire frame. No component edit was needed — all
  logic lives in the pure `renderGeneratorThumbFrame` seam.

## Context pack (brief — for S38 signal previews)

- **The seam is `renderGeneratorThumbFrame(generatorId, params, tMs, state)`** — a pure
  function: build the 26×13 `buildThumbPixelModel`, one synthetic `Trigger`, one
  `RenderContext`, call `gen.render`, read back raw RGB. No wall-clock inside; `tMs` is passed
  in (`performance.now()` from the shared ticker, or `400` under reduced motion). S38's signal
  previews can reuse this exact shape — pass a looping `tMs` and read `gen.timebase` the same way.
- **Timebase read** is `(gen.timebase ?? 'absolute') === 'voice'`, identical to
  `generator-bridge.ts` and `render.ts`. Keep these three in lock-step: any change to the
  voice-local clock/beat formula must land in all three or previews/thumbs/engine diverge.
- **The synthetic trigger** carries `drumId: 'thumb'`, `velocity: 1`, `note: 0`, stable `seq`.
  A single hit; accumulator effects fire once per stable seq (they do **not** re-spawn on the
  age wrap — see audit residuals). If a future preview wants repeating particle spawns it must
  reset `genState` at the loop seam (the thumb deliberately does not, to stay stateless).
- **`THUMB_LOOP_MS = 1600`** is exported for tests/previews that want to assert loop equality
  (`frame(t) === frame(t + LOOP)` for voice/hit-relative; `!==` for absolute).

## Visual audit — all 41 thumbnails

Programmatic audit (rendered every registered generator at looping ages 0/100/200/400/800/1200/1500,
classified lit-pixel count, animation, and loop-equality). **All 41 render; 39 animate; none render
black.** Full table archived in the commit history of the temp harness; findings:

**Fixed by this slice (fire → decay → repeat, loop-clean):** whole-drum, whole-kit, burst,
follow-hoop, lightning, radial-wash, wave-collapse, chase (voice, hoop 0 on hit). These were the
"frozen / black" thumbs; they now light on the hit and decay, repeating each loop.

**Animate on wall-clock today, will loop-restart once S26 flips them to `'voice'` (no thumb change
needed):** synced-hoops, strobe, starfield, collisions, sacred-hogs, gravity-wells, comet-trails,
orbit-rings, temp-sweep, and the trigger-hosted textures. The generic flag read means each lights up
automatically as its declaration flips.

**Free-running base/ambient (correctly stay `'absolute'`, keep phasing):** breathing-kit,
hue-rotate-kit, solid-base, plasma, rainbow-flow, tunnel, perlin-clouds, caustics, interference,
helix, spiral, lava-lamp, ripple-pond, fire, grid-glow, sidechain, swing, wipe-3d, checker-pulse.

**Residuals (animate, not frozen — cosmetic, not broken):**
- **colour-melody** & **meter-eq** — *static frame*. Input-driven (note→colour / audio-band levels)
  with no time component; the synthetic thumb has one fixed note and no audio input, so they render a
  representative constant frame. Not fixable without synthesizing fake input (out of scope); the frame
  is representative, not black.
- **confetti-burst / pixel-accum** (and to a lesser degree starfield / collisions / comet-trails) —
  *stateful accumulators*. They animate (alive), but their per-voice `genState` persists across the
  age wrap (the thumb holds one voice for the effect's life, not a new voice per loop), so they do not
  hard-restart at the loop seam. This is the known per-voice-state-reset item owned by S26's
  conversion audit; on the thumbnail it is cosmetic (the effect looks alive, just doesn't re-burst on
  the seam). A future fix would reset `genState` at the loop boundary — deliberately not done here to
  keep `renderGeneratorThumbFrame` stateless.

No thumbnail renders black or freezes at full brightness — the §B root cause is resolved.

## Deviations

- **`EffectThumb.svelte` untouched.** The spec's touch list names it, but all the behaviour lives
  in the pure `renderGeneratorThumbFrame` seam the component already calls with wall-clock `now` /
  `400`. Keeping the logic in the pure function (not the component) is what makes the acceptance a
  *pure render test* and leaves the reduced-motion / ticker / IntersectionObserver paths untouched.
  No functional gap.
- **`drumId: '' → 'thumb'` fix** (beyond the literal "looping age" ask). Without it the named
  drum-keyed effects (whole-drum, burst, radial-wash) render black, so their acceptance criterion
  could not be met. In scope (`effect-thumb-render.ts` is my surface) and regression-locked.
- **Stateful particle thumbs don't hard-restart at the loop seam** (see audit residuals). Left as an
  audit note per the spec ("audit lists remaining broken with cause"); the real fix is per-voice state
  reset, which is S26's domain and does not translate to the single-voice thumbnail model.
- **Did not edit effect impls or `registry.ts`** — sibling S26 owns those. My code reads the
  `timebase` flag generically so S26's flips light up without a thumbnail change.
