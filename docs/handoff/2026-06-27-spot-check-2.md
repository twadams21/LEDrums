## Transport recall (needs a real MIDI source + an OSC sender, voice mode)
- [ ] **Program Change** N from a DAW/controller → switches to song N in the active setlist
      (and its first section is recalled / something plays).
- [ ] **CC #0** value M → recalls section M in the active song.
- [ ] **OSC** `/ledrums/song_<n>/section <m>` (e.g. via a sender) → selects song n + section m.
      OSC works server-side without a browser.
- [ ] **CC #0 is unbindable** elsewhere: in the trigger-source editor, try to set a trigger's
      CC to 0 → it's rejected/floored with the "CC 0 reserved for section recall" hint.
- [ ] (Known gap, not a bug) the web UI's active song/section does NOT yet visually follow a
      DAW-driven recall — the engine changes, the UI highlight doesn't. Confirm it's only cosmetic.
