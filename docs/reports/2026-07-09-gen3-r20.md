# R20 — Store split 1/5: extract the monitor controller (#99)

**Branch:** `gen3r/r20-store-monitor` · **HEAD:** `8a2e710`

## What changed
First slice of the trigger-lab store split. Extracted the **PixLite controller
monitor** (S48/S49/R29) out of the 191 KB store god-file into a
constructor-injected controller, following the `SaveStatusController` prior art.

New module `apps/web/src/lib/trigger-lab/controller-monitor.svelte.ts` —
`ControllerMonitor`:
- **Runes-owned reactive state:** `status`, `candidates`, `scanning`; derived
  `takeover` getter (`status?.testPattern ?? null`).
- **Ingest methods** (server broadcasts): `ingestStatus`, `ingestDiscovery`,
  `clearOnLinkDrop`.
- **Panel send helpers:** `watch`, `discover`, `adopt`, `setAuth`, `identify`,
  `setTestData`, `backToLive` — with the same viewer-gating as before (`watch`
  is the only ungated one).
- **Injected host** (`ControllerMonitorHost`): `send(msg)`, `isViewer()`,
  `setOutput({host})`. Keeps the WS-client lifecycle, presence/role derivation,
  and the Adopt-IP output write in the store where they belong.

`store.svelte.ts` now **delegates** its public surface (API-preserving):
- `controllerStatus` — getter **+ setter** (the ui-shot `shot-seam.ts` injects a
  synthetic status by assignment, so it stays writable).
- `controllerCandidates`, `controllerScanning`, `controllerTakeover` — getters.
- `watchController`/`discoverControllers`/`adoptController`/`setControllerAuth`/
  `identifyController`/`setControllerTestData`/`backToLive` — thin forwarders.
- WS callbacks `onControllerStatus`/`onControllerDiscovery` and the `onConnection`
  link-drop clear now call the controller's ingest methods.

Reactivity crosses the getter boundary as the brief prescribed: the `$state`
lives in `ControllerMonitor`, the store exposes getters, consumers
(`ControllerStatusPanel`, `PatchControllerInspector`, `OutputPill`) read
`store.controllerStatus` etc. unchanged.

## What deliberately stayed (scope boundary)
The **engine-monitoring event log** — `monitorEvents` + filters, `serverVoices`,
`output`/`outputPacketsPerSec`, `addMonitor`/`clearMonitor`/`reportEditorFault`/
`monitorForClientMessage`/`visibleMonitorEvents` — is monitor-adjacent but
entangled with the dock voice source, the OutputPill derivations, and the
per-frame stats path. It has no dedicated seam here and the brief says leave the
ambiguous parts in place. Left for a later slice.

## Seam for R21 (undo history)
R21 dispatches next against this landed state. The undo machinery is already a
cohesive cluster on the store, distinct from what moved here:
`undoStack`/`undoLimit`/`restoringUndo`/`suppressUndoSnapshot` (fields ~706–712),
`pushUndoSnapshot`, `batchIntoCurrentUndo`, and the `restoringUndo`/`isViewer`/
`suppressUndoSnapshot` guard at the top of the snapshot path (~1124). Extract it
the same way: an `UndoHistory` controller owning the stack + flags, store
delegating `undo()`/`pushUndoSnapshot()` and the `suppressUndoSnapshot`/
`batchIntoCurrentUndo` surface via forwarders. `store.svelte.ts` still holds all
undo-related fields untouched by this slice, so R21 starts from a clean base.
