## Current anchors (verified 2026-07-05 — re-anchor by symbol, lines drift)

### Effects metadata + taxonomy (U1)
- `packages/core/src/effects/types.ts`: `EffectGenerator` gained `description?`, `tags?`,
  `deprecated?: { replacedBy; note? }` (additive — render signature unchanged).
- `packages/core/src/effects/vocabulary.ts`: **controlled tag vocab + `PlayType`** =
  `'hits'|'waves'|'particles'|'textures'|'ambient'|'meters'|'canvas'`; `playTypeForEffect()`
  first-tag-match derivation. Gallery collections and typed play nodes BOTH derive from here.
- `packages/core/src/effects/metadata.ts`: every generator described+tagged.
- `packages/core/src/effects/aliases.ts`: `EFFECT_ALIASES` (retired id → live id), consulted
  at hydrate (`normalizeGraphs`) + `buildShow`. Aliased/deprecated ids never appear in gallery.
- Flow: core registry → `apps/web/src/lib/trigger-lab/fixtures.ts` → `EffectDef` (`sim.ts`)
  now carries `description`/`tags`/`playType`/`deprecated`.
- Gallery: `EffectGallery.svelte` = collection tabs + tag chips (AND) + has-param filter +
  search(name/desc/tags), scope demoted to a chip, cards with desc clamp + tag pills +
  param-count pill. New reusable `Pill` primitive in `apps/web/src/lib/ui` + styleguide.

### Legacy pattern path DELETED (U3)
- `voice/pattern-renderer.ts`, the `Pattern` type + `pattern` field, the compositor/engine
  SoA `attrs` path, and **EffectCreator** (+ `store.createEffect`/`NewEffectInput`) are GONE.
  The voice compositor now has **ONE render path** — the generator bridge. **Custom-look
  authoring returns via the Canvas engine (U5).** 10 patterns retired via aliases; merges:
  `colour-melody`→`whole-drum` (`noteHue` bool), `burst`→`radial-wash` (`Pop` preset).

### Thumbnails (U2)
- `EffectThumb.svelte` is **generator-only**; isometric projected-dot painter
  (`apps/web/src/lib/trigger-lab/thumb-projection.ts`, ¾-angle stacked ellipses, glowing
  dots), background mini-drum for `kit-wide`-tagged effects, representative-age reduced
  motion. **Prop API (preserve it): `generatorId`, `labModel`, `params`, `w`, `h`.**

### Canvas engine (U4) — the substrate U5/U6 build on
- `packages/core/src/canvas/`: `types.ts` (`CanvasScene`, `CanvasElement` union:
  stripes/circle/gradient/polygon/checker/noise), `scene.ts` (painter-order compositor),
  `elements.ts` (6 pure element renderers), `sampler.ts` (**old `effects/field.ts`
  refactored in**; 4 samplers `hoop`/`strip`/`cylinder`/`footprint` as precomputed per-pixel
  tables), `lenses.ts` (8 lenses incl world-space `hyper4d`), `registry.ts` (scene registry;
  **`canvas:<sceneId>` ids resolve inside the existing `tryGetEffect`**), `ids.ts`.
- `GraphNode` gained `playType: PlayType` + `canvasScene?: string`; carried through
  `PlayAction`/`Voice`; hydrate infers `playType` from `effectId`.
- **NO compositor/bridge fork** — the scene engine is an `EffectGenerator` adapter;
  `compositor.ts`/`generator-bridge.ts` are byte-untouched (git-proven).
- 8 std scene params (`canvasRotDeg`, `canvasOffsetX/Y`, `canvasScale`, `samplerRotDeg`,
  `speed`, `brightness`, `hue`) via `paramSpec` → Inspector/envelopes/LFO/CC drive them
  with zero new UI. Determinism byte-identical; perf 0.536 ms/frame (5ms budget).
- **NOT yet built (U5/U6):** seed canvas scene presets (plan D4 names `canvas/presets.ts`),
  show-doc scene persistence, and all authoring UI.
