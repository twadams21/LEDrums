# R25 — Live signal previews in the Random/Note/OSC inspectors (GH #104)

**Branch:** `gen3r/r25-signal-previews` · **Status:** done, gates green.

## What changed
The Random, Note, and OSC modulation-source inspectors now show a live signal
preview above their controls, reusing the shared node-face preview component
(`NodeSignalPreview` → `SignalFace`) the way the Scope inspector embeds its
hoop preview:

- **Random** → its distribution's **density curve** (peak-normalized PDF), redrawn
  reactively as the Distribution / Steps controls change.
- **Note** → a live **gate/velocity** bar + 0–127 readout (caption follows the mode).
- **OSC** → a live **value** bar + readout.

All three ride the existing shared rAF ticker (viewport-gated; reduced-motion →
a single static frame), so reduced motion is respected via the existing
thumb-ticker behaviour with no new machinery. No new styleguide primitives were
added (the preview component already exists), so `docs/design-system.html` did
not need regenerating.

## Root cause / design notes
- The node-face `random` preview drew a hardcoded generic glyph that ignored the
  node's actual distribution. Added a pure `randomDistributionTrace(distribution,
  steps)` in `signal-preview.ts` that samples the SAME core
  `voice.sampleRandomDistribution` the engine uses (fixed-seed LCG, static —
  computed in a `$derived`, never per frame) so the drawn curve can't lie about
  what the node emits; `stepped` quantizes to its comb. `NodeSignalPreview` gained
  optional `distribution`/`steps` props; the node-face fallback glyph is unchanged.

## Follow-ups (adjacent — noted, not fixed per CONVENTIONS)
1. **`nid()` id collisions (latent bug).** Adding a node to a persisted authored
   graph can mint an id (`n-1001…`) that duplicates an existing node's id, because
   the session id counter resets on load and doesn't dedupe against persisted ids.
   Duplicate ids break select-by-id (the inspector resolves to the first match).
   This is why reaching the source inspectors for ui-shot needed the `new-graph`
   seam op (an empty graph has no colliding ids). Worth fixing `nid()` to skip
   ids already present in the loaded graphs.
2. **`scripts/with-gate-lock.mjs` breaks `pnpm gates` on >4-core machines.** It
   sets `VITEST_MAX_FORKS`/`VITEST_MAX_THREADS=4` but not the matching
   `VITEST_MIN_*`; on a 16-core box vitest's default min-workers exceeds 4 →
   tinypool `minThreads and maxThreads must not conflict`, failing before any
   test runs. Workaround used here: `export VITEST_MIN_FORKS=1 VITEST_MIN_THREADS=1`
   alongside `pnpm gates`. Fix: have the gate-lock also set `VITEST_MIN_*=1`.
