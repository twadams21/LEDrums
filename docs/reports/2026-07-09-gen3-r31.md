# R31 — Toast restyle: top-centre, larger text, per-role tint

**Branch:** `gen3r/r30-shortcuts-restyle` · **Commit:** `5a6e498`
**Gates:** `pnpm gates` green — typecheck + 1417 tests.

Trent (interactive, mid-R30): "move the toast to the top of the screen, increase the text
size, and slightly colour the background based on the toast role."

## What changed (`apps/web/src/lib/ui/ToastHost.svelte`)
- **Top-centre stack** — `.toast-host` anchored `top: var(--space-4)` (was `bottom`); the enter
  transition now descends from above (`enterY` negative) instead of rising from below. Exit fade
  and reduced-motion handling unchanged.
- **Larger text** — message `--text-xs` → `--text-sm`.
- **Per-role background tint** — a slight wash of the tone colour into the base surface plus a
  matching border, mixed in **oklab** (oklch would sweep the hue arc and muddy green/red):
  - `info` → `--accent` (lime), 10% bg / 32% border
  - `success` → `--ok` (green), 10% / 32%
  - `error` → `--live` (red), 12% / 38%
  Role still rides the icon + text too — never colour alone (accessibility contract kept).

## Design system + verification
- Styleguide toast note updated (`lib/styleguide/sections/SectionPrimitives.svelte`);
  `docs/design-system.html` regenerated (`pnpm design-system`).
- Dev-only shot-seam op added (`toast[:tone]`, `apps/web/src/lib/app/shot-seam.ts`) so ui-shot
  can capture the otherwise-transient stack; `reset()` now clears toasts between captures.
- `.ui-shots/r30x-toasts.png` (--strict, no console errors) — the three tones stacked top-centre
  with their tints and larger text.

## Deviation / follow-up
- **No "warning" tone exists** — `ToastTone` is `info | success | error` only, so warning was
  not tinted. A real amber warning is a one-line add (`ToastTone` + a `.tone-warn` wash using the
  existing `--warn` token) — flagged for Trent's call, not done unprompted.
