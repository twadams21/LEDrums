# Effects Library v2 — U5 report

Typed play nodes UI + Canvas authoring UI (issue #58). This is the handoff U6 (library fill) and
U7 (close-out) depend on: how canvas scenes are created / edited / stored, the scene-param authoring
surface, and exactly what U6 needs to register seed scenes.

## What shipped

- **Typed play palette (D3).** `TriggerGraphView` Add drawer has a **Play** group with one item per
  `COLLECTIONS` entry (Hits / Waves / Particles / Textures / Ambient / Meters / Canvas). `play` is
  removed from the generic Nodes group. Dispatch: `store.addPlayNode(playType, x, y)`.
- **playType-locked EffectGallery.** Opening the gallery from a play node pre-filters AND locks the
  pool to `node.playType` (collection rail collapses to that one collection; a `Locked · <label>`
  pill shows). The pool now draws from `store.selectableEffects` (real effects + virtual canvas
  effects). `store.pickEffect` rejects a mismatched `playType` as the authoritative backstop.
- **Node-face type chip.** `NodeCard` gained an optional `typeChip` (a tinted pill between title and
  sub). `TriggerNode` passes `collectionMeta(node.playType ?? eff.playType ?? 'ambient').label`.
  Patch nodes pass nothing, so they're unaffected.
- **Canvas Inspector.** `PlayNodeInspector` shows a scene picker for canvas nodes; a "New canvas
  scene" button creates+binds one. Canvas params render through the SAME generic param loop over
  `eff.params` (which is `CANVAS_PARAM_SPEC.map(mapParamSpec)`), so no param special-casing.
- **Objects ▸ Canvas Scenes.** New rail section with create/rename/duplicate/delete and a per-row
  **JSON editor** (day-1 authoring, locked decision 3 — no visual editor). Invalid JSON shows an
  inline error and mutates nothing; an id change is rejected.
- **Persistence + engine registration.** Scenes live in the show doc and register into the pure
  canvas registry on `setShow` so `canvas:<sceneId>` resolves in connected voice mode.
- **Clipboard portability.** Canvas scenes travel with graph/section/song ClipDoc copy-paste and are
  id-remapped on paste.

## Persistence shape

- **Web authored state:** `AuthoredState.canvasScenes?: CanvasScene[]` (`persistence.ts`, additive,
  coerced in `coerceAuthored`, no VERSION bump). Store rune `TriggerLab.canvasScenes` snapshots into
  `toAuthored()` and is assigned (even when absent → `[]`) in `applyAuthored()`.
- **Engine show:** `voice.Show.canvasScenes?: CanvasScene[]` (`packages/core/src/voice/types.ts`).
  `VoiceBusEngine.setShow` calls `syncCanvasScenes(show.canvasScenes)` FIRST — it `registerCanvasScene`s
  each and `unregisterCanvasScene`s any the previous show owned but this one drops (tracked in
  `registeredCanvasSceneIds`). Without this, connected canvas nodes render black.
- **Canvas virtual effects (NOT persisted as real effects).** Derived from scenes on demand:
  - `EffectDef.id = generatorId = canvas:<sceneId>`, `playType = 'canvas'`, `category = 'texture'`,
    `busId = 'base'`, `scope = 'kit'`, `params = CANVAS_PARAM_SPEC.map(mapParamSpec)`, tags include
    `'canvas'` (see `store/canvas-scenes.ts#canvasEffectDef`).
  - Default preset id `canvas:<sceneId>:default` (`canvasDefaultPreset`).
  - Exposed via store getters `canvasEffects`, `selectableEffects` (= real + canvas), `allPresets`
    (= real + canvas defaults, dedup by id). `effectOf` / `presetById` / `presetsForEffect` /
    `effectsForScope` all read the `selectable*`/`all*` variants so canvas nodes resolve.
  - Sent to the engine in `showSource`: `effects = [...rv.effects, ...canvasEffects]`,
    `presets = [...rv.presets, ...allPresets.filter(canvas:)]`, plus `canvasScenes`. `buildShow`
    deep-copies `canvasScenes` into the `voice.Show`.

## Scene CRUD / JSON editor

Store mutators (all viewer-guarded), in `store.svelte.ts`:

- `createCanvasScene(name?) → id` — appends `makeCanvasScene` (one drifting stripe field), `freshId('scene', …)`.
- `renameCanvasScene(id, name)`, `duplicateCanvasScene(id) → newId` (`"… copy"`).
- `deleteCanvasScene(id)` — retargets referencing play nodes onto the **first remaining** scene via
  `retargetSceneRefs` (or clears the binding if none remain), and drops saved `canvas:<id>` presets.
- `canvasSceneJson(id) → string`, `updateCanvasSceneJson(id, text) → {ok}|{ok:false,message}`.
- `setCanvasScene(node, sceneId)` — points a play node at a scene, seeds default-preset params.
- `addPlayNode(playType, x, y)` — canvas reuses `canvasScenes[0]` or creates one; other types seed
  the first non-deprecated effect of that `playType`.

JSON validation (`store/canvas-scenes.ts#parseCanvasSceneJson`): must be an object; `id` must equal
the row's id (**id is stable — duplicate to fork**); `name` non-empty; `elements` an array;
`sampler` an object. All pure + unit-tested.

## Scene-param authoring surface

- Scene picker (`Select`) + "New canvas scene" button in `PlayNodeInspector`, shown when the node's
  play type is `canvas`.
- Params: the existing generic slider/toggle loop over `eff.params`, which for a canvas effect is
  `CANVAS_PARAM_SPEC` (brightness, speed, hue, canvasRotDeg, canvasOffsetX/Y, canvasScale,
  samplerRotDeg). Preset apply/save and modulation work unchanged — a canvas node is a normal play
  node with a derived effect.

## Clipboard / Song Library portability

- `ClipDocDeps.canvasScenes?: CanvasScene[]`. Graph/section/song builders attach
  `scenesForGraphs(raw.graphs, sources.canvasScenes)` — the scene docs referenced by any canvas play
  node (`node.canvasScene` or `canvasSceneIdOf(node.effectId)`), deduped + deep-copied.
- `ClosureSources.canvasScenes?` (song-library) carries the store's scenes to the builders;
  `store.clipSources()` supplies `this.canvasScenes`.
- On paste, `remapClipDoc` builds a `sceneMap` (reuse a content-identical local scene, else
  `mint.scene()`), emits new scenes into `RemapResult.canvasScenes`, and `remapCanvasNode` rewrites
  each canvas node's `canvasScene` + `canvas:<id>` effect/preset ids. `store.applyRemapResult` unions
  the scenes and reserves their ids.
- **Known gap (follow-up, not blocking U6/U7):** the song→Song-Library path
  (`materializePaste` with `songDest:'library'`) and cross-show library-song *references* do NOT yet
  carry scene docs into the library closure. Because `showSource` sends ALL local
  `store.canvasScenes`, a referenced song's canvas nodes render only if their scene is present in the
  active show. Threading scenes through `extractSongClosure` / the resolved song-library view is the
  clean fix.

## U6 instructions (seed scenes + gallery exposure)

U6 fills the day-1 library. For **canvas seed scenes** the engine seam already exists (U4); U5 adds
the authoring layer. To expose a seed scene:

- **Where to put seeds.** Register core seed scenes at module load in
  `packages/core/src/canvas/` (e.g. a `presets.ts` that calls `registerCanvasScene(scene)` for each,
  imported by `packages/core/src/index.ts`). This makes `tryGetEffect('canvas:<id>')` resolve for
  built-in scenes everywhere WITHOUT them living in the user's show doc. This is the recommended
  home (parallel to how generator effects are registry-backed built-ins).
- **Do NOT** seed them into web fixtures as user `canvasScenes` — those are user-authored, persisted,
  and deletable; seeds should be built-in + non-deletable like the generator effects.
- **Gallery exposure.** The gallery lists `store.selectableEffects` filtered by `playType`. Built-in
  canvas seeds need a virtual `EffectDef` with `playType:'canvas'` in that pool. Two options:
  1. add the built-in scenes' `canvasEffectDef`s to the seed `EFFECTS` list (fixtures) so they appear
     as normal canvas effects (simplest; then a Canvas node's locked gallery shows the seed library);
  2. or extend `store.canvasEffects` to also project `listCanvasScenes()` (core registry) built-ins.
     Prefer (1) for gallery parity unless you want core to own the whole seed set.
- Helper to reuse: `store/canvas-scenes.ts#canvasEffectDef(scene)` builds the `EffectDef`;
  `canvasDefaultPreset(scene)` the default preset. `collectionMeta('canvas')` gives the label/blurb.
- Lens presets (D5) are authored inside a scene's `lenses[]` array — seed scenes can ship lens chains
  directly; no new UI needed (the JSON editor already round-trips `lenses`).
