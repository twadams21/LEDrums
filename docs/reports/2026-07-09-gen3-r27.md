# R27 — Protected-anchor header for Output/Trigger inspectors (GH #106)

**Branch:** `gen3r/r27-anchor-header` · **Status:** done

## What changed
The Output node inspector no longer renders a kind selector with no valid
option. `output` is a protected graph anchor (like the trigger root) and is
excluded from `KIND_OPTS` (`node-options.ts`), so when an output node was
selected the shared node header showed a `Select` whose value (`output`) was
not among its options — an empty selector — plus a remove button the store
refuses to honour (`removeNode` guards `isAnchorNode`). It now gets a proper
title-block header instead.

- **New primitive** `lib/ui/AnchorHeader.svelte` — an inspector title block for
  a protected anchor: optional accent-tinted leading icon + `h3` title, a mono
  sub-line, and an optional trailing action snippet. Same scale/rhythm as the
  existing patch / trigger `ihead` headers, so anchors read as first-class
  inspector titles.
- **OutputNodeInspector** renders `AnchorHeader` (CircleDot, `--role-output`
  tint, "Output" / "graph output — every layer lands here") above its Scope /
  Target fields.
- **Inspector.svelte** (dispatcher) gets an `output` branch *before* the shared
  kind-selector branch, so output no longer falls through to the empty selector;
  the now-dead `output` arm inside the shared branch was removed. Non-anchor
  nodes are untouched — they still hit the shared header with the kind `Select`.
- Trigger anchor was already correct: it renders its own `ihead` header via
  `TriggerSourceInspector` and never showed an empty selector — left as-is.
- **Styleguide**: `AnchorHeader` demoed in `SectionPrimitives` (Output + Trigger
  variants); `docs/design-system.html` regenerated.
