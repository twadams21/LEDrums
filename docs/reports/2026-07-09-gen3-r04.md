# R04 — Auto-wire a new Effect to Output (+ toast, single undo) · GH #83

## What changed
Adding an Effect node now auto-wires it to the terminal Output anchor, so it makes light on
the next hit instead of sitting silent. The auto-wire:
- is **one undoable action** — a single Ctrl/Cmd+Z reverts the add **and** the wire together;
- routes through the **validated `connect` path** (`classifyConnection`), so an impossible wire
  (never expected for a fresh Effect → Output) is skipped silently rather than toasting an error;
- **announces itself** with one plain-language info toast (reusing R02 toast conventions).

The behaviour lives in the store (`addNode`), the single seam every UI add-path flows through
(`TriggerGraphView`, palette drops, the shot seam).

## How
- **Batched undo.** Added a private `suppressUndoSnapshot` flag + `batchIntoCurrentUndo()` helper
  (mirrors the existing `restoringUndo` guard). `addNode` pushes ONE undo snapshot as before, then
  the auto-wire runs `connect()` inside `batchIntoCurrentUndo` so `connect`'s own
  `pushUndoSnapshot` is suppressed — the wire folds into the add's checkpoint. Undo stack gets one
  entry, not two.
- **Scope.** Only the light-making Effect node auto-wires (`node.kind === 'effect'`, which both
  the `play` and `effect` add-kinds resolve to). Modifier / envelope / lfo / mix / source nodes do
  not. No-op (silent) when the graph has no Output anchor.
- **Toast copy:** *"Effect wired to the Output anchor — it lights on the next hit."* (`tone: info`).
