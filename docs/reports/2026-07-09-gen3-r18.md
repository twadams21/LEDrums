# R18 — Render-plan compile caching keyed by structure signature (GH #97)

**Branch:** `gen3r/r18-plan-caching` · **Status:** done · determinism-preserving, no UI change.

## What changed
`compileRenderPlan` ran on every trigger fire (`evalGraphGen3` and the delay-drain
`evalGraphGen3From`), so a fast roll recompiled the graph per hit. R18 caches the
compiled plan across hits, invalidated by a structure signature — object identity
alone is unsafe because the store mutates the graph in place (same object, changed
structure), so an identity check can't see a structural edit.

New in `packages/core/src/voice/render-plan.ts`:
- **`renderPlanSignature(graph)`** — a string over *exactly* what `compileRenderPlan`
  reads to build a plan: each node's `id`/`kind`/`y` and each edge's
  `id`/`from`/`to`/`toPort` (plus node/edge counts + version as a cheap prefix). It
  deliberately excludes params (effectId, mode, mixBlendMode, thresholds, …): compile
  never reads them, and eval reads them off the live node objects the plan references.
  Under in-place mutation a param edit mutates those nodes without replacing the graph,
  so the cached plan keeps reading current params — only a *structural* edit must
  invalidate.
- **`RenderPlanCache` / `createRenderPlanCache()`** — a `WeakMap<TriggerGraph, {signature,
  plan}>` cache. Reuse is gated on graph identity **and** signature: same object + same
  signature → return the cached plan; in-place structural edit → signature moves →
  recompile; a distinct graph object → its own entry (a multi-pad kit never thrashes a
  single slot, and dropped graphs are GC'd). Exposes `compileCount` (misses, for
  tests/telemetry) and `reset()`.

Wiring:
- `eval-graph.ts` — `EvalState` gains an optional `renderPlanCache`. A `compilePlan(state,
  graph)` helper routes both compile sites through the cache when present, else a fresh
  compile. Absent cache ⇒ identical behaviour (only the compile is skipped on a hit, never
  the plan's content), so eval stays pure/deterministic given (time, inputs, model).
- `engine.ts` — engine-owned `renderPlanCache` (like `mixMemberSnapshots`), injected in
  `evalState()`, `reset()` in `setShow` alongside the other live-state clears (authored
  graphs are replaced wholesale there).

## Tests
- **Signature:** stable for an unchanged graph; moves for all 7 structural-edit classes
  (add/remove node, change kind, move node y, add/remove edge, retarget flow→mod);
  unmoved by param-only edits.
- **Cache:** reuses the same plan object with no recompile (identity + `compileCount`);
  invalidates on each structural-edit class (in-place mutation); reuses across a
  param-only edit; keys per graph (distinct graphs don't evict each other); `reset()`
  drops plans.
- **Parity (acceptance):** identical eval actions with and without the cache across an
  8-hit fast roll (one compile served all 8), and across a mid-roll structural edit
  (exactly one recompile forced).
