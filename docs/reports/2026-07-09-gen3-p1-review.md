## The six review invariants — each verified against the code

**1. One validator, no bypasses.** ✔ Every wiring path routes through
`classifyConnection` / `classifyReconnect` (`store/graph-wiring.ts`):
- `connect` / `reconnect` (store) return the `WireRejection | null` from the classifiers;
  `canConnect` / `canReconnect` are thin `=== null` wrappers (`graph-wiring.ts:143,157`).
- Auto-wire (R04) calls `this.connect(...)` (`store.svelte.ts:2860`) — same seam.
- Splice (R08) `canSplice` calls `classifyConnection` twice on the without-edge graph
  (`graph-wiring.ts:133-134`); `spliceOnDrop` re-wires via `this.connect` (`store.svelte.ts:3007-3008`).
- In-drag validity (`validateDrop`, `TriggerGraphView.svelte:522`) is a **read-only mirror**
  over `canConnect` — `predictParamPort` replaces the auto-exposing `paramPortFor` so hover
  never mutates. No parallel validator anywhere.

**2. Undo shapes.** ✔
- R04 add+wire = **one** entry: `addNode` pushes one snapshot (`store.svelte.ts:2814`),
  the auto-wire runs inside `batchIntoCurrentUndo` so `connect`'s own push is suppressed
  (`store.svelte.ts:2860`).
- R08 position + splice = **separate** entries: the view commits position first via
  `moveNode` (own checkpoint), *then* `spliceOnDrop` pushes its own snapshot
  (`TriggerGraphView.svelte:450-456`, `store.svelte.ts:3004`). One Ctrl/Z pops the wiring,
  a second reverts position — covered by `store.splice.test.ts`.
- Batching composes safely: `batchIntoCurrentUndo` saves/restores the previous
  `suppressUndoSnapshot` in a `finally` (`store.svelte.ts:1152-1160`), so a splice nested
  in any suppressed flow can't corrupt the stack. Verified by fuzz + the splice suite.

**3. Lint — one model, two surfaces, no contradiction.** ✔ Strip and badges read one
derived list: `lintDisplay = lintEntries(lintIssues)`; badges index it via
`lintIndex.set(lintEntriesByNode(lintDisplay))` (`TriggerGraphView.svelte:415-416`) — they
cannot drift. The three passes are mutually exclusive by node kind: `no-path-to-output`
targets producers (effect/play), `dead-branch` targets producerless carriers
(scope/mix/output), `empty-scope` targets scope/output nodes *reached by* a producer —
a dead-branch node has no producer upstream so is never empty-scope, and vice-versa
(`reachability-lint.ts:24-33`, `scope-lint.ts:126-131`). Copy is plain + one imperative
action per code (`graph-lint.ts:27-44`).

**4. Cache interaction.** ✔ Reachability reads structure only (kinds + flow edges), so it
can never go param-stale in the R18 plan cache — stated and true (`reachability-lint.ts:16-19`,
signature note `render-plan.ts:155-161`). Empty-scope's param-staleness is confined by the
`renderPlanSignature` contract ("do not consume `issues` through `RenderPlanCache` for UI"),
and the UI honours it: the lint derive compiles **uncached** via `voice.compileRenderPlan(g)`
(`TriggerGraphView.svelte:412`). The cache (`createRenderPlanCache`) is engine-eval-only.

**5. Locked in-drag interaction contract (instant, no motion).** ✔ `.wire-invalid`
(red/dotted/dull) and `.edge-splice-armed` (accent/thick/glow) are pure static style rules —
no `transition`/`animation` (`GraphCanvas.svelte:243-247,283-287`). Node lint badge is
instant (no motion on the `forNode` render). The lint **strip** has a gentle enter-fade that
collapses under `prefers-reduced-motion` (`GraphLintStrip.svelte:48-95`) — correct: the strip
is a panel surface, not an in-drag/node-interaction state, so the locked contract doesn't bind it.

**6. DEV-only preview runes don't leak to prod.** ✔ `wireInvalidPreview`,
`spliceArmedPreview`, `lintPreview` are all read only under `import.meta.env.DEV`
(`TriggerGraphView.svelte:337,385,410`); the shot seam installs `__LEDRUMS_SHOT__` only under
DEV (`shot-seam.ts`). Dead-code-eliminated in production builds.

## Follow-ups (non-blocking)

1. **`nit` — R07 has no reachability inspector row.** R06 added an empty-scope callout to
   `OutputNodeInspector`; R07's `no-path-to-output` / `dead-branch` surface only as strip +
   badge. ACs don't require an inspector row; the R07 author flagged the parity gap. Candidate
   for a later polish pass. (`OutputNodeInspector.svelte`)
2. **`nit` — empty-scope cache-staleness is a documented latent footgun.** Correct today
   because every lint UI compiles uncached, but a future dev routing `issues` through
   `RenderPlanCache` would silently serve stale empty-scope badges after a param-only edit.
   Guarded only by the `renderPlanSignature` comment — worth a runtime assertion or a typed
   "structural-issues-only" accessor if the cache ever feeds UI. (`render-plan.ts:155-161`)
3. **`nit` — a zero-displacement drag-then-splice leaves a no-op position undo entry.**
   `moveNode` always pushes a snapshot, so if a spliced node is dropped with ~no net movement
   the "second undo reverts position" step is an empty undo consuming a keystroke. A splice
   inherently requires dragging onto a wire, so net movement is near-guaranteed — cosmetic at
   most. Pre-existing `moveNode` behaviour, not introduced by R08. (`store.svelte.ts:onDragStop`)
