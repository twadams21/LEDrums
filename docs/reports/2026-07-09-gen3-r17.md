# R17 — Web sim imports core graph/action types; double-cast removed (GH #96)

**Branch:** `gen3r/r17-sim-core-types` · **Status:** done · behaviour-preserving, no UI change.

## What changed
The web sim (`apps/web/src/lib/trigger-lab/sim.ts`) mirrored core's evaluation
types (`PlayAction`, `StopAction`, `PendingAction`, `Action`, `PlayDraft`,
`MixInputDraft`, `TriggerCtx`) as local declarations and bridged them to the core
evaluator — which has been the runtime eval path since R16 — with `as unknown as`
casts. The mirrors and casts are gone:

- Deleted the local mirrored declarations. Replaced with type-only aliases to
  `@ledrums/core`:
  - `type PlayAction = voice.PlayAction`
  - `type Action = voice.Action`
  - `type PlayDraft = voice.PlayDraft`
  - `export type TriggerCtx = voice.EvalTriggerCtx` (kept exported so the many web
    files that import `TriggerCtx` from `../trigger-lab/sim` are unaffected).
  - `StopAction` / `PendingAction` were only used inside the local `Action` union;
    they collapse into `voice.Action` and are dropped. `MixInputDraft` was already
    referenced only as `voice.MixInputDraft`, so no local alias is needed.
- Removed three now-redundant bridge casts, all in `Sim`:
  1. `evalGraph()` — `voice.evalGraph(...) as Action[]` → direct return.
  2. `enqueueCorePending()` — `descriptor.draft as PlayDraft | null | undefined` →
     `descriptor.draft` (already core `PlayDraft`).
  3. `drainPendingFires()` — the `voice.evalChildren as unknown as (…) => Action[]`
     structural cast → a plain typed call (its signature already matches).

Type drift between the sim and the core evaluator it delegates to now fails to
compile.

## Files touched
- `apps/web/src/lib/trigger-lab/sim.ts` — mirrors → aliases; 3 casts removed (+16/−59).
- `packages/core/src/voice/index.ts` — add a **type-only** `PlayDraft` re-export to
  the `voice` barrel (it was already `export`ed from `eval-graph.ts`, just not in the
  barrel). Type-only; core purity (no Node/DOM/IO) untouched.

## Deviation
The ticket noted "web only." One line was added to the core `voice` barrel — a
type-only re-export of the already-exported `PlayDraft` — because the sim needs
`voice.PlayDraft` and deriving it locally (`Omit<voice.PlayAction, …>`) would
re-introduce exactly the kind of formula-mirror the ticket asks to remove. The
change adds no runtime coupling and does not touch core purity.
