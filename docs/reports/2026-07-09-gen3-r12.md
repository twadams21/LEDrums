# R12 — Canvas drag-over highlight for add-node drags; grip icon on graph rows (GH #91)

Two interaction affordances that make dragging explicit: the Trigger canvas
announces itself as a live drop target while a node is dragged in from the Add
pane, and section graph rows expose a real grip handle instead of a whole-row
grab cursor.

## What changed
1. **Canvas drop-target ring.** While a new node is dragged from the Add pane
   over the Trigger graph canvas, the surface wears an accent ring + faint wash —
   the canvas analogue of the Sections reorder target (R11 `.col.section-target`),
   so "drop is live" reads at a glance in one visual language. Clears on drop and
   on leaving the canvas. The ring is an overlaid, `pointer-events: none` element,
   so it never intercepts the drop. Instant (no motion), per the locked
   interaction contract.
2. **Grip icon on graph rows.** Each section graph row gains a leading
   `GripVertical` handle in a dedicated left gutter. The grab cursor is confined
   to the grip; the rest of the row keeps its default (select) cursor. The grip is
   faint at rest and brightens with the row on hover. Row content (Workflow icon +
   labels) is unshifted — the grip lives in an added `padding-inline-start` gutter,
   absolutely positioned.

## ui-shot state (drag-only → preview rune)
The highlight only exists during a live HTML5 drag, which headless Chrome can't
drive — so it's reached the same way R03/R08/R11 reach their drag-only states: a
tiny DEV-only preview rune (`canvas-drop-preview.svelte.ts`) the shot seam pins,
merged into the real drag state under `import.meta.env.DEV` (dead-code-eliminated
in prod). New seam op `canvas-drop`. **No `shots.json` entries** (ad-hoc captures
only, per conventions).

## Deviations / follow-ups
- Scope confined to the **Trigger** canvas: it is the only add-from-pane drop
  target (`ADD_NODE_DRAG_TYPE`). Patch graph has no palette drop, so no ring there.
- Grip added to `SectionGraphRow` only — the draggable graph rows. `GraphRow.svelte`
  (Objects view) is not draggable, so it gets no grip. None blocking.
- Design system unchanged: no new reusable styleguide component (the ring reuses
  R11's token language inline; the grip is a one-row affordance), so `pnpm
  design-system` was not regenerated.
