# R06 — Node lint badges + empty-scope on node face and Output inspector (GH #85)

**Branch:** `gen3r/r06-node-lint-badges` · **Status:** done

## What changed
Second slice of the lint surface (after R05's strip). Two things: (1) every lint
finding that names a node now also rides that node as a **corner badge** — one
lint model, two surfaces (strip + badge) that always agree; (2) a new
**empty-scope** compiler finding — a scope chain that can never light a pixel —
is surfaced on the node face and as an actionable row in the Output inspector.

### Core — the empty-scope lint (the real work)
R05 noted its three codes can't occur in authored graphs, so empty-scope is the
first genuinely user-reachable lint. It is computed as **core truth**, pure and
unit-tested.

- **New `packages/core/src/voice/scope-lint.ts`** — `detectEmptyScopes(nodesById,
  flowChildrenById)`. Walks each effect/play node's stream forward exactly as
  eval does (a Scope/Output node narrows the scope, a Mix resets it to whole-kit),
  accumulating a **source-independent** static scope. A Scope/Output node is
  flagged only when it is reached by ≥1 stream and **every** reaching stream is
  empty there — a single live stream keeps it off the list.
- **Provably-empty only.** A bare (auto/firing-drum) target is a wildcard drum
  that can coincide with any concrete drum at runtime, so it never proves an
  empty intersection on the drum axis — only concrete disjoint drums or disjoint
  hoop indices do. Mirrors `scope.ts`'s runtime `intersectScopeTargets` with
  `sourceDrumId` treated as a wildcard, so the lint is free of false positives on
  the common "Auto (firing drum)" targets. The running intersection (not pairwise)
  catches three-way hoop cases.
- **`render-plan.ts`** adds `'empty-scope'` to `RenderPlanIssueCode` and appends
  the findings in `compileRenderPlan`. **Non-fatal** by design (a dead branch
  doesn't break the render; other branches still light) — kept out of the `fatal`
  set.

### Web — badges + inspector row (one lint model, two surfaces)
- **`graph-lint.ts`**: `PROBLEM`/`ACTION` gain empty-scope copy ("Scope never
  matches" / "Widen this scope or the one upstream — they don't overlap, so
  nothing here lights"). New pure `lintEntriesByNode(entries)` groups the anchored
  findings by node id.
- **`graph-lint-index.svelte.ts`** (new): a reactive `GraphLintIndex` handed to
  the xyflow custom nodes via context (same pattern as `GraphHover`), kept in sync
  from the **one** compiled issue list — so strip and badges can't drift.
- **`TriggerNode.svelte`**: reads `forNode(id)` and renders a warn-toned corner
  badge (top-left, opposite the drum-link badge) with a tooltip naming the problem
  + fix, plus a faint warn border wash. **Instant, no motion** (locked node
  interaction contract).
- **`OutputNodeInspector.svelte`**: a warn empty-scope callout row above the
  Scope/Target fields, reusing the shared lint copy so strip, badge, and inspector
  state the finding identically.
- **`shot-seam.ts`**: an `empty-scope` ad-hoc state that authors a real offending
  graph (effect drum:A → Output drum:B) so ui-shot reaches all three surfaces from
  genuine `compileRenderPlan` output.

### Design system
No regeneration: the lint badge is a **TriggerNode-local node-face marker**, like
the existing drum-link badge and modifier-count chip (neither in the styleguide),
not a new reusable primitive. The warn callout reuses design tokens + the strip's
row idiom.

## Deviations / notes
- The lint badge is added on the standard NodeCard path only; a bands-switch node
  caught in a flow cycle would not get a corner badge (the strip still shows it).
  Acceptable minor gap — empty-scope (this ticket's new lint) targets scope/output
  nodes, which always use the NodeCard path.
- Coordinate-free rule honoured: no changes to AddPalette, sections rows, or sim's
  type declarations (R17/R12 territory).
