## Part 1 — Carried nit fixes

### Nit 3 — dead legacy Block-tree evaluator deleted · `c829436`
`apps/web/src/lib/trigger-lab/sim.ts`. The `Sim` class carried a legacy tree-walking eval
path (`trigger` / `evaluate` / `resolveParams` / `switchIndex` / `modeWord`) unreachable since
R16 routed all sim evaluation through core's Gen3 `evalGraph` (`triggerGraph → evalGraph →
voice.evalGraph`). Repo-wide grep: **zero** call sites of `Sim.trigger(`; `evaluate` only
self-recursed. Deleted the island (−97 lines). Kept the block→graph compiler (`treeToGraph`,
`Block` types — still live via store seeding) and the shared `modeGlyph`. No tests deleted;
86 sim tests green.

### Nit 2 — zero-displacement `moveNode` undo guarded · `8f891e4`
`store.svelte.ts:moveNode`. `moveNode` always pushed an undo snapshot, so a drag (or a
splice-drop onto a wire) that ended where it started left an empty position checkpoint — Ctrl/Z
after a splice popped that no-op before reaching the splice wiring (P1 review follow-up #3, rooted
in `moveNode`, not `spliceOnDrop`). Now snapshots only when `x`/`y` actually changed; position +
live-position writes stay (harmless no-ops when unchanged). Added a splice test asserting a
zero-displacement drop leaves the splice as the only new checkpoint (non-tautological: without the
guard the second undo would still find the node present).

### Nit 4 — node/edge id mints collision-safe · `65cf71f`
`store.svelte.ts` + `store/hydrate.ts`. The `nid` counter resets to its base on reload, so a bare
`nid('n')`/`nid('e')` could re-mint an id a persisted/pasted node already carries — a duplicate id
silently breaks select-by-id (inspector resolves to the first match), the latent bug R25 flagged.
Routed the node/edge mint sites (`addNode`, `addPlayNode`, `placeClone`, `addModifierNode`,
`connect`, and the env-map migration in `hydrate`) through the existing `freshId(prefix, exists)`
helper, deduping against the graph's own ids. Section/show/song mints unchanged (own reserve paths).
Added `store.id-collision.test.ts` (2 tests).

### Nit 1 — reachability inspector row · `b31b7e3`
R07's `no-path-to-output` / `dead-branch` findings surfaced only on the lint strip + node badge —
no inspector row, unlike R06's empty-scope callout (P1 review follow-up #1). Added a reusable
`ui/LintCallout.svelte` primitive (styleguide-registered) + a `nodeLintEntries(issues, nodeId,
codes?)` helper, and wired a warn row into every inspector where reachability anchors:
`PlayNodeInspector` (no-path-to-output), `ScopeNodeInspector` + `ContainerNodeInspector`/mix and
`OutputNodeInspector` (dead-branch). Refactored R06's bespoke empty-scope block to the shared
primitive (identical markup; Output now also shows dead-branch). Copy stays the shared lint table,
so strip / badge / inspector read identically. Tests: `nodeLintEntries` unit tests + an Output
dead-branch render test; R06 tests updated to the generic `.lint-callout` selectors. **ui-shot**
confirms the "Not reaching Output" row on the Effect inspector (`.ui-shots/r15-reachability-
inspector.png`). Design system regenerated with the Lint callout demo.

**Note:** the empty-scope live ui-shot didn't render its callout because only the web dev server
was run (no WS backend → no live kit-drum data, which empty-scope's scope-intersection needs). The
refactored empty-scope render path is proven by the passing `OutputNodeInspector` unit test.

`packages/core/src/voice/render-plan.ts:40,49,86,132`. `RenderPlan.planNodesById` is built on

**SF2 — Two independent empty-scope determinations that can visibly disagree.**
`apps/web/src/lib/app/docks/inspectors/scope-inspector.ts:142-158` (`effectiveScopeForNode`) vs
`packages/core/src/voice/scope-lint.ts:98` (`detectEmptyScopes`). `ScopeNodeInspector.svelte:98-102`

drum** (`drums[0]?.id ?? 'kick'`, `scope-inspector.ts:146`) — and it ignores the drum selected in the
inspector. The canonical R06 empty-scope lint resolves with a **firing-drum wildcard**, flagging only

kick but lights on snare shows "Effective: Empty — No LEDs after this Scope" in the inspector while the
lint (correctly) keeps it live and shows no warning — a user-facing contradiction between two surfaces
that answer the same question. Compounded by R15: `ScopeNodeInspector` filters its lint row to
`['dead-branch']` only (`ScopeNodeInspector.svelte:36`), so the reconciling `empty-scope` badge never
appears there. The P1 review verified the strip↔badge and scope-lint↔scope.ts twins but did not
cross-check this inspector readout against the lint — the cross-surface seam a scoped review misses.

sim.ts` vs `packages/core/src/voice/types.ts`: `PlayMode` (`sim:64`==`types:20`), `Scope`

**N2 — `BlockKind` name collision across sim/core.** `sim.ts:75` (`play|all|random|sequence|switch|

will mislead.

**N3 — Redundant lint recompute.** `GraphLintStrip.svelte:17` runs `lintEntries(issues)` again over
issues the parent `TriggerGraphView.svelte:415` already derived as `lintDisplay`. Cheap, but the strip
and the badge index compute `lintEntries` over the same issues twice per derive; pass the derived list.

**N4 — Sole wall-clock read in core (`perfNow`).** `packages/core/src/voice/engine.ts:695,722`
(helper `:813`) calls `performance.now()` inside `tick()`. It feeds **only** `this.perf.tickMs`
(observability), never the frame buffer, and is guarded (`typeof performance !== 'undefined'` → 0 in a
non-DOM host), so `frame()` stays pure. Not a determinism violation against the AGENTS.md render-loop
rule — noted only because it is the one wall-clock read in `packages/core`.

**N5 — Raw hex in `ScopeHoopPreview`.** `ScopeHoopPreview.svelte:90,141` uses `#0a0d12` (canvas
fill + CSS background) rather than a token, unlike sibling canvas previews (`NodeSignalPreview`,
`NodeStatePreview`, `ParamRowTick`) that read tokens with hex only as a JS fallback. Likely deliberate
(no exact token for that backdrop; the rest of the file is tokenized) — low priority.

**Observation —** `blockChildren` (`sim.graph-compilation.ts:24`) is `export`ed with zero consumers
outside its own file; drop the `export`.
