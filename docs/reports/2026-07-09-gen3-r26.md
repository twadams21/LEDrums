# R26 тАФ Inspector label rows тЖТ shared Field primitive (GH #105)

**Branch:** `gen3r/r26-field-migration` ┬╖ **Status:** done, gates green.

## What changed

Migrated the hand-rolled inspector "label row" implementations (a `<span class="k">`
label column + control, each re-deriving `--field-label-col` / 2xs / weight-500 label
styling) to the shared `Field` primitive (`layout="row"`). One rhythm, one implementation.

**Migrated (4 inspectors, 13 rows):**
- `DelayNodeInspector` тАФ Mode (segmented), Time (unit `ms`), Division.
- `LfoNodeInspector` тАФ Wave, Rate (segmented), Freq (unit `Hz`), Division, Phase (unit live `%`).
- `ContainerNodeInspector` тАФ No-repeat (toggle), On, Threshold (slider), Invert (toggle), Chance (slider), Blend.
- `OutputNodeInspector` тАФ Scope (segmented), Target.

**Field primitive extension** (`apps/web/src/lib/ui/Field.svelte`): added an optional
`unit` prop тАФ a short trailing readout that renders in its own grid column **outside**
the control (`ms` / `Hz` / a live `%`). This was required for pixel-faithfulness: the
hand-rolled rows put the unit beside the box, which `CommitInput`'s inner `suffix`
(unit *inside* the border) can't reproduce. Row-layout only; ignored in stack.

**Design system:** styleguide Field-row demo now shows the `unit` variant (numeric `ms`
+ slider live `%`) and the note explains `unit` vs `suffix`. `docs/design-system.html`
regenerated in the same change (AGENTS.md non-negotiable).

## Deliberately out of scope (not the clean label-row rhythm)

Documented rather than force-fit, to stay mechanical/behaviour-preserving:
- **`ScopeNodeInspector`** тАФ its `.k` labels are **weight 600** (a distinct local rhythm
  shared by its two-line toprow and read-only readouts); migrating just the Drum row to
  Field (weight 500) would be a visible regression and break intra-inspector consistency.
- **`PlayNodeInspector`** тАФ rows are bordered section-bars with trailing action clusters
  (scene/preset buttons) + a param table (`.prow`, 84px labels); a distinct idiom, and
  piecemeal migration would leave `.k` half-used.
- **`ModifierNodeInspector`** тАФ param table + a two-control toolbar (`.bar` + bypass).
- **Container `.bandrow` / `.mixctl`** тАФ multi-control rows (number chip + comparator +
  slider + delete; mono layer name + slider) on custom grids; the bare valuemode
  segmented has no label.
- **`SectionInspector` look-rows** (icon-in-label тАФ Field's `label` is a string) and
  read-only recall-rows.
- **`PatchDrumInspector`** origin/rotation (3├Ч xyz inputs + `<em>` unit label, stack).
- **`PatchControllerInspector`** `.checkrow` (control-first toggle, reversed order).
- **`ControllerStatusPanel` `.auth`** тАФ a stack label+control+hint candidate, but its
  label warms colour conditionally (`.needs`), which Field doesn't model; also R29 turf.
- **`LayersDock` `.xfade`** тАФ content-width label, not `--field-label-col`; migrating
  would widen the label column (regression).

These would each need a further Field variant (label snippet / conditional label colour /
trailing-action slot) тАФ good follow-ups if the rhythm is to be pushed further, but beyond
a behaviour-preserving sweep.
