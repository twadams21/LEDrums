## Summary

U6 fills the day-1 Effects Library v2 surface with authored Canvas content and four gap-fill Gen-3 native effects.

The implementation keeps the U4/U5 architecture intact:

- Canvas scenes are authored `CanvasScene` data.
- Built-in scene docs are registered through the existing core canvas registry and render through `canvas:<sceneId>` / `tryGetCanvasEffect`.
- Web gallery exposure projects the same scene docs into normal `EffectDef` cards and default presets.
- New native effects use the existing emission contract (`createEmitterState` + `updateEmissions`) and do not fork the compositor or voice bridge.

## Web gallery / preset inventory

Web file: `apps/web/src/lib/trigger-lab/fixtures.ts`

The web fixture registry now adds `CANVAS_EFFECTS`, projected from `BUILTIN_CANVAS_SCENES`:

- `EffectDef.id = canvas:<sceneId>`
- `EffectDef.generatorId = canvas:<sceneId>`
- `playType = 'canvas'`
- `busId = 'base'`
- `scope = 'kit'`
- params come from `CANVAS_PARAM_SPEC`
- every built-in canvas effect gets a default preset via the existing `PRESETS` seed path

Compatibility note: U5's `setCanvasScene()` picker currently binds canvas play nodes only to scene docs present in `store.canvasScenes`. To make the built-in gallery cards selectable in a blank show without editing U5's large store seam, `seedAuthored()` now seeds `canvasScenes: structuredClone(BUILTIN_CANVAS_SCENES)`. This is intentionally documented for U7 verification: the cards are selectable today, but a later hardening pass could make built-ins read-only/non-deletable in Objects instead of treating them as seeded editable templates.

## New Gen-3 native effects

Registry file: `packages/core/src/effects/registry.ts`

Four new gap-fill native effects were added and registered:

| id | file | collection | contract |
|---|---|---|---|
| `orbit-comet` | `packages/core/src/effects/impl/orbit-comet.ts` | Particles | emission-based, per-drum, hoop-aware, seeded |
| `scan-plane` | `packages/core/src/effects/impl/scan-plane.ts` | Waves | emission-based, 3D kit-wide airspace sweep |
| `drum-sonar` | `packages/core/src/effects/impl/drum-sonar.ts` | Waves | emission-based, per-drum hoop sonar rings |
| `gravity-drops` | `packages/core/src/effects/impl/gravity-drops.ts` | Particles | emission-based, per-drum falling bead/trail |

Each effect:

- uses `createEmitterState` and `updateEmissions`;
- spawns once per trigger seq;
- layers concurrent hits as independent emissions;
- has deterministic payload derivation from trigger data;
- emits finite clamped channels through the existing framebuffer path;
- carries D1 metadata directly on the generator (`description`, controlled-vocabulary tags).

## Review fixes (local review pass, 2026-07-05)

CI was red on the original push; a local review pass fixed:

- **typecheck**: `seedAuthored()` assigned `structuredClone(BUILTIN_CANVAS_SCENES)` (readonly array type) to the mutable `canvasScenes` — now clones per-scene into a fresh mutable array.
- **`registry.test.ts` catalog count** not bumped for the 4 new natives (45 → 49; the 4 ids added to the presence list).
- **`gravity-drops` dead-zone bug**: with `trail` (default 0.22) smaller than a drum's hoop gap (1/3 on a 4-hoop drum) the falling bead could sit between hoops and light nothing (visible flicker on sparse-hoop drums; also the failing test). The trail reach now widens to at least 1.1× the drum's hoop gap.
- Restored `fixtures.ts` rationale comments (S18/S19 param-mapping notes, generator scope rationale, seed-tree notes) that the U6 diff had stripped.
- Added 3 more scenes: `aurora-veil`, `spiral-galaxy` (chained lenses), `prism-fan`.
