## Changes

`packages/core/src/voice/render-plan.ts`
- Removed `planNodesById`, `outputId`, `incomingFlowEdgesById` from the
  `RenderPlan` interface and the `compileRenderPlan` return.
- Kept the local `outputId` and `incomingFlowEdgesById` computations — they
  still feed `detectUnreachable` and the `missing-output` issue; only the
  exposed fields are gone.
- Dropping `planNodesById` orphaned its entire supporting taxonomy, all of
  which existed solely to populate that field, so they were pruned too:
  `RenderPlanNode` interface, `nodeCategory()` function, `RenderPlanNodeCategory`
  type. (`RenderPlanChild`, `RenderPlanIssue`, `isFlowEdge` are still consumed
  by eval/lints and were retained.)
- **Untouched (per brief):** `renderPlanSignature` semantics and the cache
  invariant comment — the signature already lists only the structural fields
  compile reads, none of which changed.

`packages/core/src/voice/render-plan.test.ts`
- Removed the `nodeCategory` taxonomy test (covered dropped code) and the
  `.outputId` / `.planNodesById` / `.incomingFlowEdgesById` assertions in the
  boundary-anchor test. Modulation-side-channel exclusion is still asserted via
  the retained `flowChildrenById.get('env') === []`.

## Deviations / judgment calls

- **Pruned the `nodeCategory` taxonomy, not just the field.** `nodeCategory` is
  an exported, unit-tested function, but its only production purpose was
  building the now-dropped `planNodesById`; after the drop it was reachable
  only from its own test. Removing it (and `RenderPlanNode`/
  `RenderPlanNodeCategory`) is the honest completion of the prune, not a
  restructure — no behavior, signature, or cache change. If the taxonomy is
  wanted as future public API, revert just the `render-plan.ts` taxonomy hunk;
  the field-drops stand independently.
