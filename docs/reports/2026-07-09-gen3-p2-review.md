## AC verification

**Search (R09).** `filterAddGroups` (add-pane.ts) filters the *whole*
vocabulary — iterates every group, matches `name` **or** `hint`
(case-insensitive, trimmed), drops empty groups, returns `[]` for a blank query
so the caller falls back to browse. `AddPalette` renders flat grouped results
via the shared `previewButton(it, group.key)` snippet, so a result adds with
**its own** category key (covered by AddPalette.test.ts). No regression from
R10's tile restyle: the searching branch and the browse (chip) branch are
mutually exclusive in the same component. ✔

**DnD affordances contract (R11/R12).**
- *No prod leak.* Both preview runes (`canvasDropPreview`,
  `sectionsDndPreview`) are consumed only under `import.meta.env.DEV`
  (`TriggerGraphView.svelte:97`, `SectionsView.svelte:42`). `shot-seam` is
  dynamically imported and DEV-gated at the call site (`App.svelte:27-29`), so
  the whole seam — and the rune setters — dead-code-eliminate from prod
  bundles. `reset()` clears both runes (shot-seam.ts:108,112). ✔
- *Overlay can't intercept the drop.* The R12 canvas ring is a separate
  `.drop-ring` element with `pointer-events: none; z-index: 4`
  (TriggerGraphView.svelte:660-670), and the lint overlay above it is also
  `pointer-events: none`. Drop handlers live on `.gwrap`
  (`ondragover/leave/drop`) and are correctly guarded — `onPaletteDragOver`
  early-returns unless `dataTransfer.types` includes `ADD_NODE_DRAG_TYPE`, so
  it never `preventDefault`s unrelated drags, and xyflow node drags (pointer
  events, not HTML5 DnD) don't collide. `dragActive` clears on leave and drop;
  `onPaletteDragLeave` only clears when the pointer truly left `canvasWrap`. ✔
- *Instant highlight / no lift-click motion.* R12 ring and R11 section-target
  are instant (no animation). One deviation — the R11 insertion line has a
  120ms enter — see should-fix S1. ✔ (with S1 caveat)

### Should-fix
**S1 — insertion-line enter animation vs the locked "instant highlight"
contract.** `SectionColumn.svelte:205-212` (`.insert-line` /
`@keyframes insert-line-in`). The 2px line animates `scale: 0.6 1 → 1 1` +
opacity over `--dur-120`. Because the line unmounts/remounts per `dropIndex`
(`{#if dropIndex === i}`), dragging a row slowly down a column replays the
grow-in every time the pointer crosses into the next gap — a repeated
motion where the Phase-2 DnD contract and R12's ring/section-target both
specify *instant* highlight. Low severity: `prefers-reduced-motion` disables
it, and it's a deliberate `/make-interfaces-feel-better` choice per the R11
report. Flagging for Trent's ruling — if the instant-highlight contract is
strict, drop the enter (or keep opacity, no scale) so R11 matches R12.

### Nit
**N1 — dead `class:drop-active` on the canvas wrapper.**
`TriggerGraphView.svelte:563` toggles `class:drop-active={dropActive}` on
`.gwrap`, but no CSS rule in this component targets `.drop-active` (the only
`.drop-active` rule lives scoped inside `SectionColumn` and, by Svelte style
scoping, cannot reach this element). The real affordance is the `.drop-ring`
overlay. The binding is vestigial (likely a leftover from a
border-on-wrapper approach superseded by the overlay). Harmless but
misleading — remove it.
