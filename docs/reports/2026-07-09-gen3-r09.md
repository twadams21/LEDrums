# R09 — Add pane search across all node categories (GH #88)

## Outcome
The Add pane now carries a search field that filters the **whole** node
vocabulary. While a query is active, results render as a flat list still
grouped by category (same `NodeCard` previews as browse); clearing the query
restores the existing two-stage category browse. Reuses the shared
`SearchField` primitive from `lib/ui/`.

## What changed
- **`add-pane.ts`** — new pure `filterAddGroups(groups, query)`: case-insensitive,
  whitespace-trimmed match on each item's `name` **and** `hint`; returns groups
  in original order carrying only matching items, empties dropped; `[]` for a
  blank query so the caller falls back to browse.
- **`AddPalette.svelte`** — a `SearchField` at the top of the pane; `query` +
  `searching`/`results` derived state. When searching, renders the flat grouped
  results; otherwise the unchanged Stage-1 tiles + Stage-2 browse. Extracted a
  shared `previewButton(it, groupKey)` snippet so search results and browse
  render identical cards/drag/add behaviour (add/drag now take an explicit
  `groupKey`, so a result adds with **its own** category key). Search-mode empty
  state is plain text ("No nodes match …"), consistent with R10's empty-state
  direction and not touching the browse empty state / category tiles (R10 scope).
- **`shot-seam.ts`** — new `search:<query>` op (the sanctioned way to teach
  ui-shot new state). The query is `AddPalette`-local, so the op drives the real
  `input[aria-label="Search nodes"]` and dispatches `input` (like `openSettings`
  drives the TopBar button), rather than a store method.
- **`scripts/ui-shot/shots.json`** — `gen3-add-pane-search` preset
  (`view:trigger,search:s`, target Node editor).
- **`docs/design-system.html`** — regenerated (the styleguide demos `AddPalette`
  live, so the search bar now shows there too).
