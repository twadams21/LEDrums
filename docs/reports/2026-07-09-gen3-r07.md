# R07 — Reachability lint (GH #86)

Third and final slice of the lint surface. A pure structural reachability pass in
core feeds two new non-fatal render-plan issue codes; the strip copy and node
badges come free from the R05/R06 reactive pipeline (no new UI machinery).

## What changed
- **`packages/core/src/voice/reachability-lint.ts`** (new) — `detectUnreachable(...)`, a
  pure pass over the compiled flow graph emitting two codes:
  - `no-path-to-output` — a layer **producer** (`effect`/`play`) whose forward flow
    never reaches the Output anchor (the unwired / torn-off render leaf).
  - `dead-branch` — the **head** of a branch that reaches Output but has no producer
    upstream (a scope/mix/output wired toward the anchor with nothing to render).
    Anchored to the topmost layer node of each producerless branch, so a chain lights
    one badge, not one per node. Requires the node to be wired (≥1 incoming flow edge),
    so a bare unwired Output is *not* flagged.
- **`render-plan.ts`** — added both codes to `RenderPlanIssueCode`; wired
  `detectUnreachable` into `compileRenderPlan` (non-fatal, like `empty-scope`); gated on
  an Output existing so `missing-output` (fatal) stays the single message when there is
  no anchor. Extended the `renderPlanSignature` invariant note: reachability reads
  structure only (kinds + flow edges), so unlike `empty-scope` it can **never** go
  param-stale in the R18 plan cache.
- **`graph-lint.ts`** — `PROBLEM`/`ACTION` copy for both codes
  ("Not reaching Output" / "Wire this into the Output node…"; "No layer to render" /
  "Wire an Effect or Play into this branch…").
- **`shot-seam.ts`** — one adapter method `notReachingOutput()` (op `no-path-to-output`):
  reuses/authors an Effect, severs its outgoing flow, selects it. Captures the real R07
  surface end-to-end from genuine `compileRenderPlan` output.

## Design notes
- Producers = `effect`/`play`. Flow-chain layer carriers = `scope` (transform) + `mix`/
  `output` (collectors). `modifier` and modulation sources sit **off** the flow (they wire
  to `mod`/`param` ports, never `in`), so they are neither producers nor carriers here.
- Rules 1 and 2 target disjoint kinds (producers vs. transform/collector) → no double
  badge. A producerless scope/output is unreachable by the `empty-scope` walk (which starts
  from producers), so the two lints are mutually exclusive in practice.
- Torn-apart graphs read correctly: delete the last wire (fx→scope→∅) → fx gets
  `no-path-to-output`; delete the first wire (∅→scope→output) → scope gets `dead-branch`.
