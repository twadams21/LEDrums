# R16 — Delete the legacy pre-Gen3 eval path (GH #95)

**Branch:** `gen3r/r16-delete-legacy-eval` · **Commits:** `85df69c` (core), `34b42d7` (web)

## Outcome
One evaluator is the only evaluator. Deleted the dead legacy (pre-Gen3, `version !== 3`)
graph-eval path in `packages/core` and its hand-copied web-sim mirror. Strictly no behaviour
change — the Gen3 path already handles everything live.

## Deadness verification (traced before deleting)
- **Core:** `engine.setShow()` is the *only* graph ingestion and normalizes every graph via
  `normalizeTriggerGraphToGen3` (engine.ts:262). So `resolved.graph.version === 3` always holds
  at `evalGraph`/`evalChildren` — the `version !== 3` branch was unreachable in production.
- **Web:** the store normalizes graphs to Gen3 on hydrate (`store/hydrate.ts`) and in
  `show-builder.ts`, so every graph reaching `sim.triggerGraph` at runtime is already Gen3.
- The legacy path was kept alive **only by tests** feeding legacy-shaped fixtures (no `version`,
  no terminal `output` node). No non-test call sites; the deleted core helpers were all private
  (never exported — `index.ts` still exports `evalGraph`/`evalChildren`/`bandIndex` unchanged).

## What changed
**`packages/core/src/voice/eval-graph.ts`** (859 → 543 lines; −327/+11)
- `evalGraph` / `evalChildren` now dispatch straight to the Gen3 path.
- Deleted private legacy helpers: `evalNode`, `evalNodeWithDraft`, `evalValueSwitch`,
  `draftFromUpstream`, `mixInputsFor`, `childrenOf`, `childrenViaPort`, `incomingFlowEdges`.
- Kept shared helpers used by the Gen3 path: `bandIndex`, `edgeOpacity`, `freezeRandomMappings`,
  `modeWord`, `switchIndexN`, `nodeStateKey`, `appendLabel`. **Gen3 evaluator itself untouched**
  (per the R13 sibling note — delete only).

**`apps/web/src/lib/trigger-lab/sim.ts`** (892 → 803 lines; −300/+19)
- `sim.evalGraph` now normalizes any legacy graph to Gen3 and delegates to `voice.evalGraph` —
  exactly as the real engine does at `setShow`, so offline preview and live output share one path.
- Deleted the ~230-line mirror: `evalNode`, `draftFromUpstream`, `mixInputsFor`, `evalValueSwitch`,
  `childrenOf`/`childrenViaPort`/`incomingFlowEdges`/`edgeOpacity`, `resolveNodeParams`,
  `switchIndexN`, `freezeRandomMappings`; simplified `drainPendingFires`' legacy branch away
  (descriptor graphs are always Gen3, produced by the core evaluator that enqueued them).
- Removed the now-unused `bandIndex`/`GraphNode` imports.

**Total production eval code removed: ~627 lines** (327 core + 300 web).

## Tests
- **Core:** migrated the 6 tests that fed legacy-shaped fixtures onto the canonical Gen3 shape
  (added `version: 3` + terminal `output` node + leaf→output edge) so they keep asserting live
  behaviour — `eval-graph.playtype.test.ts` (×2), `modifier-graph.test.ts` (×1),
  `modulation-graph.test.ts` (×3). Full `src/voice` suite: **321 passed**.
- **Web:** no test migration needed — legacy-shaped sim fixtures now flow through
  `normalize → Gen3` and pass untouched. Sim + store/bridge suites: **123 passed** (79 sim,
  44 store/bridge).

## Deviations / follow-ups
- **Out of scope (noted, not touched):** `sim.ts` also carries a separate, older `Block`-tree
  evaluator (`trigger(tree)` / `evaluate(block)` / `switchIndex` / `resolveParams`) with no live
  or test call sites found. It's the pre-*graph* model, not the pre-Gen3 *graph* mirror this
  ticket targets — a candidate for a later dead-code sweep.
- **Harness bug (not mine):** `pnpm gates` sets `VITEST_MAX_FORKS/THREADS=4` but no `MIN`, so on
  ≥8-core machines vitest's default `minThreads` (~cpus/2) exceeds the cap and Tinypool throws
  `minThreads and maxThreads must not conflict`. Worked around by pinning
  `VITEST_MIN_FORKS=1 VITEST_MIN_THREADS=1`. The gate lock (`scripts/with-gate-lock.mjs`) should
  also default the MINs.
