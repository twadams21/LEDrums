# R08 — Wire-splice on node drop, pre-release arming + splice-only undo (GH #87)

**Branch:** `gen3r/r08-wire-splice` · **Status:** done, gates green.

## What changed

Dragging a node over an existing flow wire now **arms a splice**: the wire lights
accent + glows before release (pre-release indication). Releasing wires
`source → node → target`. The splice wiring is its **own undo entry recorded after**
the drag's position commit, so one Ctrl/Cmd+Z pops the wiring while the node stays
where it was dropped.

### Store seam (source of truth)
- **`canSplice(graph, edgeId, nodeId)`** (`store/graph-wiring.ts`, pure) — a dropped
  node may splice into an edge only when it is a plain trigger-**flow** wire (not a
  `mod` chain or a `param:` modulation wire — "insert in the middle" is meaningless
  there), the node is neither endpoint, and **both** resulting wires
  (`source →(source-port) node`, `node → target (target-port)`) are legal. Validated
  on the graph *without* the spliced edge so the replaced wire never trips the
  dup/cycle guard. Total — never throws.
- **`TriggerLab.spliceOnDrop(edgeId, nodeId)`** — removes the edge and re-wires
  through the validated `connect` path, preserving the source band/output port and the
  target input port. Its **own `pushUndoSnapshot`**; the remove + two connects fold
  into that one checkpoint (`batchIntoCurrentUndo`), so undo reverts the whole splice
  at once. Announces success with one info toast. No-op (returns false) when invalid.
- **`TriggerLab.canSplice(edgeId, nodeId)`** — read-only mirror the view arms on.

### View (arming + drop)
- **`splice-geometry.ts`** (pure) — `edgeUnderNode(...)` hit-tests the dragged node's
  rect against each wire's source-right→target-left chord and returns the nearest
  overlapped edge id. Unit-tested; no DOM/xyflow.
- **`TriggerGraphView.svelte`** — during `onNodeDrag` (single-node drags), arms the
  overlapped wire when `store.canSplice` agrees; `onNodeDragStop` commits the position
  first (its own undo entry) then `spliceOnDrop`. The armed edge id re-decorates the
  edge array (`edge-splice-armed` class).
- **`GraphCanvas.svelte`** — `.edge-splice-armed`: accent stroke, thicker,
  `drop-shadow` glow. Instant, no motion (locked graph-interaction contract).

### ui-shot seam
- **`wire-preview.svelte.ts`** — added `spliceArmedPreview` rune (mirrors
  `wireInvalidPreview`); the armed state is drag-only, so the shot seam pins it and the
  view (DEV-only) arms its first flow wire. Dead-code-eliminated in production.
- **`shot-seam.ts`** — `splice-armed` state op + reset clear.

## Deviations / notes
- Splice restricted to **flow wires** (not mod/modulation) — matches the spec's
  "source→node→target" flow semantics; non-flow shapes are rejected naturally by the
  two-connection validation anyway.
- Multi-node drags never arm (ambiguous which wire to splice into).
- **Design system:** no new reusable styleguide component added (the armed state is a
  transient graph-canvas edge treatment, not a design-system primitive), so no
  regeneration was needed.
- **Pre-existing (out of scope):** `TriggerGraphView.svelte` carries an unused
  `import { kindIcon, kindLabel, tint }` from `./trigger-node-meta` — untouched (not
  mine to fix; noted for a cleanup pass).
