# R11 — Sections DnD: insertion line + section-target outline (GH #90)

## Outcome
Dragging a graph row now shows an accent **insertion line** at the hovered gap;
reordering a section shows an accent **column outline** on the drop target. Both
indicators are predictable before release and clear on drop/cancel. ui-shot states
added for both.

## What changed
- **Insertion line (row drag):** while a graph row is dragged, the target column
  renders a 2px accent bar in the exact gap the row would land in. The gap is
  resolved geometrically from the pointer Y vs each row's vertical midpoint, so the
  line tracks the nearest gap (top of a row → before it, past the midpoint → after).
  The gap index is an original-list index, which is exactly what
  `moveGraphPlacement` consumes (it handles the source-removal shift itself).
- **Section-target outline (section reorder):** while a section column is dragged,
  the hovered column gets a bright accent ring + faint wash, distinct from the
  merely-active column's thin border.
- **Clear on drop/cancel:** all indicator state lives in `SectionsView` and is reset
  by `clearDrag()` (wired to `dragend` on both drag sources — fires on successful
  drop and on cancel) and by the drop handlers. A `dragleave` on the columns region
  clears a lingering indicator when the pointer leaves the columns mid-drag.
- **Drag handling centralised:** dragover/drop moved off the per-row/`onDragOver`
  props onto a single column-level handler that dispatches by drag kind and computes
  the gap. `SectionGraphRow` is now purely a drag *source* (+ a `data-graph-row`
  marker for the geometry); rows no longer carry dragover/drop handlers.
- **ui-shot:** the indicators only exist during a live HTML5 drag, which headless
  Chrome can't drive. Added a dev-only rune (`sections-dnd-preview.svelte.ts`) that
  the screenshot seam pins so the states render statically. Two shot-seam ops
  (`sections-insert`, `sections-reorder`) + two presets
  (`gen3-sections-insert-line`, `gen3-sections-reorder-target`). The preview is
  merged into `SectionsView` only under `import.meta.env.DEV`, so it is inert /
  dead-code-eliminated in production.

## Polish (/make-interfaces-feel-better)
- Insertion line: negative vertical margins collapse the flex gap so the 2px bar
  sits *in* the gap (no layout shift); rounded ends + accent glow; a quick
  scale-X/opacity enter via `--dur-120`/`--ease-control`, disabled under
  reduced-motion.
- Section-target ring uses layered `box-shadow` (crisp 1px + soft 4px) over a border
  colour so it reads apart from the active-column border and adapts to the surface.
