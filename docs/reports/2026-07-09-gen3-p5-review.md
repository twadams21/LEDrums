## Spec-dimension checks (all pass)

1. **Previews truthful.** `randomDistributionTrace` samples the SAME core
   `voice.sampleRandomDistribution` + `quantizeSteppedRandom` the engine draws
   with (`eval-graph.ts:205–206`); the preview's `clamp01` before quantize is a
   no-op since every distribution branch already returns [0,1]
   (`modulation.ts:131–157`), so the drawn curve can't diverge from the emitted
   value. Note/OSC live bars read core `sampleNote`/`sampleOsc` on the sim
   tables (`store.svelte.ts:3732–3740`).
   - **No per-frame compute in `$derived`.** Density is a single
     `$derived` (`NodeSignalPreview.svelte:46`), recomputed only on
     distribution/steps change, never per frame.
   - **Shared ticker, viewport-gated, reduced-motion → static frame.** All
     previews ride `SignalFace` → the shared `effect-thumb-ticker` (no
     per-preview rAF), IntersectionObserver-gated, `prefers-reduced-motion` →
     one static frame (`SignalFace.svelte:34–89`).
   - **No leaks.** Both effects return teardown: `ticker.subscribe` returns its
     unsubscribe; the observer effect returns `observer.disconnect`.
   - Confirmed `Sim.timeMs`/`ccTable`/`oscTable`/`noteTable` are plain
     fields/Maps (not `$state`, `sim.ts:290,342–348`), so reduced-motion
     previews genuinely freeze at mount (as R25 claims) while normal motion
     stays live via the ticker's own rAF — no hidden reactive redraw loop.

2. **Field migration behaviour-preserving.** `unit` is row-only — `hasUnit`
   requires `layout === 'row'` and is dropped in stack (`Field.svelte:24,30`);
   it adds a third grid column only when present (`.row.has-unit`,
   `Field.svelte:48`), so non-unit rows keep the original 2-column rhythm. The 4
   migrated inspectors carry no leftover hand-rolled label rows: 0 `--field-
   label-col` re-derivations remain; Container's lone `class="k"`
   (`ContainerNodeInspector.svelte:76`) is the `≤` comparator glyph inside the
   `.bandrow` multi-control idiom R26 explicitly documented as out of scope, not
   a half-migrated label. Out-of-scope inspectors (Scope/Play/Modifier/Section/
   Patch*) genuinely differ in rhythm as the report states.

3. **AnchorHeader / dispatcher ordering.** The `output` branch
   (`Inspector.svelte:119`) precedes the shared `{:else if node}` kind-selector
   branch (`:123`) and matches on the specific `node.kind === 'output'`, so it
   can't shadow another kind and output can't fall through to the empty
   selector. `output` is excluded from `KIND_OPTS` (`node-options.ts:59`).
   Remove-button absence is anchor-only: output and trigger branches omit the
   `Trash2` IconButton; every non-anchor branch keeps kind selector + remove
   (`removeNode` also guards `isAnchorNode`).

4. **Design system.** Styleguide has the Field `unit` variant (numeric `ms` +
   live `%`, note distinguishing `unit` vs `suffix`) and AnchorHeader (Output +
   Trigger) demos (`SectionPrimitives.svelte`). No NodeIconChip in the previews
   (canvas-drawn), so nothing else to document.

5. **A11y.** Previews are not canvas-only in meaning: `SignalFace` carries an
   `aria-label` naming each preview, and every source's semantic identity is
   also in adjacent labeled controls (the Distribution `Select`, the Note/OSC
   fields). See nit N1.

## Nits / observations (non-blocking)

- **N1 — live numeric readout is `aria-hidden`.**
  `NodeSignalPreview.svelte:193` marks the note/osc/cc value readout
  `aria-hidden="true"`; AT users get the labeled canvas ("Note live value") but
  not the current number. This is deliberate and consistent with the
  pre-existing CC preview (a per-frame-changing value as `aria-live` would spam
  a screen reader), so it's arguably correct — flagged only for the record, no
  change requested.
- **N2 — `randomDistributionTrace` runs 65 536 synchronous samples**
  (`signal-preview.ts:158`) on the main thread each distribution/steps change.
  Cheap in practice (discrete, interaction-triggered, sub-millisecond) and not
  per-frame; sample count could drop by 4× with no visible histogram change if
  it ever shows up on a profile. Not worth acting on now.
- **N3 — cross-ticket coexistence (flagged per brief, not re-reviewed).**
  `OutputNodeInspector` now hosts R27's `AnchorHeader` above R06's empty-scope
  `role="status"` row; they compose cleanly (header, then body).
  `OutputNodeInspector.test.ts` is 4 tests (R27's 2 + R06's 2), all green. R13's
  Mix copy is untouched by R26 (Container change was structural markup only, per
  its report). No conflicts.
