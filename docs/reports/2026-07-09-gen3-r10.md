# R10 — Add pane category tiles get node-style icon chips; empty state de-carded (GH #89)

Two visual fixes in the Trigger graph's Add pane so it reads as one coherent
surface in the node visual language.

## What changed
1. **Stage-1 category tiles carry the node icon chip.** Each category (Effect ·
   Route · Modulate · Modify) now shows the same tinted icon-with-wash chip that
   rides every node card — so a category reads as part of the node language, not
   a separate control. Icon/tint are declared per group in the taxonomy
   (representative node glyph + role tint).
2. **Stage-2 empty prompt de-carded.** "Select a node category." lost its dashed
   card box (min-height, centered, tinted fill) and now renders as plain
   left-aligned text — a prompt, not an empty box presented as content.
   Consistent with the search no-match text.

## Shared primitive (use-or-extend)
The tinted chip was duplicated visual language living inline in `NodeCard`.
Rather than add a third copy, extracted **`NodeIconChip.svelte`** (icon + tint +
`size`) and refactored `NodeCard` to use it (visually identical: 30px chip / 16px
glyph). The category tiles use it at `size=26`. Added a styleguide demo card
("Node icon chip") in `SectionComposites` and regenerated `docs/design-system.html`.
