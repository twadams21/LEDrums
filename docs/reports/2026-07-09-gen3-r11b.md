# R11b — Sections DnD polish: drag visuals + vertical section insert-line (Trent feedback on R11 #90)

## Outcome
Three Trent-flagged issues on the shipped R11 Sections DnD are fixed. Dragging now speaks
one accent-**green** language: the active column no longer flashes a cyan/blue selection
ring, a graph row drags as a clean compact card (not a broken double), and section reorder
draws a **vertical insert-line in the gap between columns** instead of washing the target column.

## Root causes + changes

### 1. Cyan/blue "focus-ish" ring on the active column (the wash Trent disliked)
Root cause was **not** a focus ring — it was `color-mix(in oklch, var(--accent) 45%,
var(--border))` on `.col.active`. oklch interpolates the **hue arc**, so mixing lime (128°)
with the blue-grey border (256°) passes through **cyan (~205°)** — the active-section border
rendered cyan. Fixed by mixing in **oklab** (rectangular, no hue arc) so it stays a muted
accent-green. Separately, removed the `.graphlist.drop-active` per-column background wash +
inset outline, so a graph drag shows **only** the insert-line, not an accent wash on every column.

### 2. Broken dragged-graph-row ghost (double row, stray ✕)
The native HTML5 ghost snapshotted the whole row — grip, status dot, hover-revealed ✕ —
semi-transparent over the still-visible original. Fixed in `SectionGraphRow`:
- A compact **custom drag image** (Workflow icon + graph name only, no ✕) rendered off-screen
  and handed to `setDragImage` on dragstart.
- The **source row dims** (`opacity: 0.4`, no layout jump) while it is the drag source, so
  there's one clear "this is moving" representation.

### 3. Section reorder: vertical insert-line instead of target highlight
Replaced the `.col.section-target` ring/wash with a **vertical accent insert-line** pinned in
the gap between the columns the section would land between — the vertical twin of R11's
horizontal graph line, same stroke/glow/120ms enter.
- New pure geometry `columnGapIndexAt(cols, clientX)` (horizontal twin of `gapIndexAt`),
  sharing an internal axis resolver.
- Section dragover/drop moved to the **`.cols` container level** so hovering the empty
  inter-column gaps (not just a column) resolves a target; the gap index feeds `moveSection`
  (which already accounts for source removal).
- The line is an absolutely-positioned overlay whose x/top/height are derived from the column
  rects (no layout shift; reaches the leading/trailing gaps too).
- The preview rune's `section` variant now carries `{ index }` (gap) instead of `{ sectionId }`;
  `shot-seam` pins an interior gap.
